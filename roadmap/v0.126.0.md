# v0.126.0 — Per-endpoint federation credentials (FEAT-03)

## Theme

OAuth2 Bearer and API-key credentials per remote SPARQL endpoint, stored
encrypted via `pgcrypto pgp_sym_encrypt`, injected transparently at query time.

## Deliverables

- [x] `_pg_ripple.federation_credentials` table with `endpoint_iri`, `auth_type`,
      `encrypted_token`, `header_name`, `created_at`, `rotated_at`
- [x] `pg_ripple.federation_credential_key` GUC (SECURITY SENSITIVE — never
      logged, never visible via `SHOW`)
- [x] `pg_ripple.set_federation_credential(endpoint_iri, auth_type, token)` SQL
      function (SECURITY DEFINER) — encrypts token via `pgp_sym_encrypt` before
      insert/update
- [x] `pg_ripple.rotate_federation_credential(endpoint_iri, new_token)` — atomic
      UPDATE with `rotated_at = now()` for zero-downtime rotation
- [x] `pg_ripple.federation_credential_audit()` — returns
      `TABLE(endpoint_iri TEXT, auth_type TEXT, token_age_days FLOAT8,
      last_used_at TIMESTAMPTZ)` without exposing plaintext tokens
- [x] Federation HTTP call path in `src/sparql/federation/http.rs` decrypts
      credential and injects `Authorization: Bearer …` or custom API-key header
      per-request; plaintext never cached
- [x] SSRF guard validates endpoint URI *before* credential lookup to prevent
      credential-oracle SSRF attacks
- [x] `GET /federation/{endpoint}/auth-status` HTTP endpoint (write-auth
      required) returning credential age and auth type
- [x] Migration script `sql/pg_ripple--0.125.0--0.126.0.sql`
- [x] 10 pg_regress tests

## Exit Criteria

- All 10 pg_regress tests in `v0126_federation_credentials.sql` pass
- Zero credential plaintext appears in `_pg_ripple.federation_credentials`
- SSRF bypass prevention: credential lookup only after `resolve_and_check_endpoint()`
- Migration chain: `pg_ripple--0.125.0--0.126.0.sql` applies cleanly
