# v0.96.0 — A15 Medium: Performance, Code Quality, Test Coverage (Full Details)

**Theme:** Assessment 15 Medium remediation — performance improvements, code quality
refactoring, and test coverage gaps.

**Status:** Released

**Scope:** Large

**Prerequisites:** v0.95.0

## Deliverables

### M15-05: HTAP tombstone-skip optimisation

Maintain a `tombstone_count BIGINT NOT NULL DEFAULT 0` column in
`_pg_ripple.predicates`. When the merge worker truncates tombstones, it sets
`tombstone_count = 0`. Rebuild the HTAP view to skip the `LEFT JOIN` /
`EXCEPT` when `tombstone_count = 0`, eliminating the join overhead on the hot
read path for predicates with no pending deletes.

**Migration:** `sql/pg_ripple--0.95.0--0.96.0.sql` adds the column and
rebuilds affected views.

### M15-06: Star-pattern self-join collapse in `sparql/optimizer.rs`

Detect `(?s p1 ?o1 . ?s p2 ?o2 . …)` star shapes in BGP patterns — same
subject variable bound across multiple triple patterns — and emit a single
subject-seeded CTE instead of N separate VP-table joins. Gate behind
`pg_ripple.star_join_collapse` GUC (default: `on`).

### M15-11: Separate federation connect timeout GUC

Add `pg_ripple.federation_connect_timeout_secs` (default: 10) as a separate
GUC from `pg_ripple.federation_timeout`. The connect timeout covers the
TCP/TLS handshake; the existing `federation_timeout` remains as the query-body
timeout. Applies to all ureq agent builders.

### M15-17: `pagerank_with_writes.sh` concurrent-load test

Add `benchmarks/pagerank_with_writes.sh`: 4 pgbench writers inserting random
triples + 1 read-only SPARQL query + 1 background PageRank run, all
concurrently for 60 seconds. Assert PageRank converges without deadlock and
SPARQL returns consistent results.

### M15-21: Verify cyclic-group pre-check source in parallel Datalog

Add a pg_regress test (`tests/pg_regress/sql/datalog_cyclic_parallel.sql`) that
defines two mutually-recursive rules (A depends on B, B depends on A),
materialises them with `pg_ripple.datalog_parallel_workers = 2`, and asserts
that the result is correct. This exercises the cycle-merge path (P13-06).

### M15-22: Arrow Flight `EXPLAIN` row-estimate path

Remove the `COUNT(*)` fallback from `pg_ripple_http/src/arrow_encode.rs`.
The `EXPLAIN (FORMAT JSON, ANALYZE FALSE)` path introduced in v0.91.0
(HTTP-04) is now the sole row-estimate mechanism. If EXPLAIN fails, log a
warning and skip the row-count guard rather than falling back to a full
`COUNT(*)` scan.
