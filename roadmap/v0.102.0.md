# v0.102.0 — What-If Reasoning (Hypothetical Inference)

**Theme**: Run inference on hypothetical facts without touching real data.

> **In plain language:** Sometimes you want to ask "what *would* happen if I added this new
> fact?" — before actually adding it. A cardiologist might ask: "If I record that this patient
> takes drug X, would my drug-interaction rules infer a dangerous combination?" A compliance
> officer might ask: "If we classify this transaction as high-risk, what downstream conclusions
> would that trigger?" This release lets you pose those questions in a temporary sandbox:
> assert hypothetical facts, run inference, see what would be derived or retracted — then
> discard it all. The real database is untouched.
>
> **Foundation document**: [plans/expert-system.md](../plans/expert-system.md)
>
> **Effort estimate: 3–4 person-weeks**

## Prerequisites

- v0.101.0 released and all tests passing.

## Deliverables

### `hypothetical_inference()` SQL Function

- [ ] `pg_ripple.hypothetical_inference(hypotheses JSONB, rules TEXT DEFAULT 'default') → JSONB`
  - `hypotheses` format: `{"assert": [{"s": "...", "p": "...", "o": "..."}], "retract": [...]}`
  - Runs inference in an isolated sandbox using the overlay mechanism (see below)
  - Returns `{"derived": [{"s":"...", "p":"...", "o":"..."}], "retracted": [{"s":"...", "p":"...", "o":"..."}]}` — the diff of what *would* change
  - Does not write to any VP table or `_pg_ripple.derivations`; all changes exist only in session-local temporary objects
  - `rules` selects which Datalog rule set to run (defaults to the `'default'` rule set)

### Overlay Mechanism

- [ ] For each predicate referenced in `hypotheses`, create session-local temporary tables:
  - `_hyp_assert_{predicate_id}` — rows to add (same schema as the VP table)
  - `_hyp_retract_{predicate_id}` — rows to remove
- [ ] For each such predicate, create a session-local view `hypothetical_vp_{predicate_id}` defined as:
  ```sql
  (SELECT * FROM _pg_ripple.vp_{predicate_id}
   EXCEPT ALL SELECT * FROM _hyp_retract_{predicate_id})
  UNION ALL
  SELECT * FROM _hyp_assert_{predicate_id}
  ```
- [ ] Inference engine reads `hypothetical_vp_{predicate_id}` views in place of real VP tables for predicates that have an overlay; non-overlaid predicates continue to use real VP tables directly — no query rewriting in the VP scan layer required
- [ ] All temporary tables and views dropped automatically on `ROLLBACK` or session end (PostgreSQL's natural `ON COMMIT DROP` / session-end cleanup)
- [ ] `pg_ripple.hypothetical_max_assertions` GUC (INT, default: 10000) — raises PT0450 if `hypotheses` would assert more rows than this limit, preventing inadvertent sandbox creation that exceeds session memory

### Error Catalog

- [ ] PT0450: `hypothetical_inference: hypothesis set exceeds pg_ripple.hypothetical_max_assertions limit (%d)`

### REST Endpoint

- [ ] `POST /hypothetical` in `pg_ripple_http` — request body: `{"hypotheses": {...}, "rules": "default"}`, response: `{"derived": [...], "retracted": [...]}`

### Tests

- [ ] `tests/pg_regress/sql/v0102_hypothetical.sql`:
  - **Isolation**: run `hypothetical_inference()`, confirm no rows appear in real VP tables after the call
  - **Rollback correctness**: begin a transaction, call `hypothetical_inference()`, rollback — assert no temp tables remain
  - **Correct diff**: load a base graph, assert a hypothetical fact that triggers a known rule, confirm the expected derived triple appears in `derived` and not in the real graph
  - **Retraction**: hypothetically retract a fact, confirm a previously-inferred triple now appears in `retracted`
  - **Side-effect absence**: call `hypothetical_inference()` twice with different hypotheses in the same session — assert sessions are clean between calls
- [ ] PT0450 is raised when `hypothetical_max_assertions` is exceeded

### Migration & Control

- [ ] Migration script `sql/pg_ripple--0.101.0--0.102.0.sql` — comment-only (no schema changes; overlay tables are session-local)
- [ ] `pg_ripple.control` updated to `default_version = '0.102.0'`

## Exit Criteria

- `cargo pgrx regress pg18` passes all tests including `v0102_hypothetical`
- `hypothetical_inference()` returns a non-empty `derived` array when the hypothetical facts trigger a known rule
- No rows appear in any real VP table after `hypothetical_inference()` returns
- Rollback leaves no residual temporary objects
- PT0450 is raised when the assertion limit is exceeded
- `POST /hypothetical` HTTP endpoint returns a valid JSON diff
