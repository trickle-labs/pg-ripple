# v0.127.0 — pg_tide Relay Migration Cleanup

## Theme

Remove the remaining current-tense pg-trickle relay assumptions and make pg_tide
the canonical relay, outbox, and inbox transport throughout pg_ripple.

## Deliverables

- [x] `pg_ripple.relay_available()` SQL helper for relay/outbox/inbox checks
- [x] Deprecated `pg_ripple.trickle_available()` retained as a compatibility alias
      for relay availability
- [x] CDC bridge runtime gate moved from `has_pg_trickle()` to `has_pg_tide()`
- [x] CDC bridge trigger function changed from dynamic table inserts to
      `tide.outbox_publish(outbox_name, payload, headers)`
- [x] `_pg_ripple.cdc_bridge_triggers.outbox_name` catalog column added while
      preserving `outbox_table` as a compatibility alias
- [x] pg_tide tested/bundled version bumped to `0.33.0`
- [x] Relay docs, examples, runbooks, and Docker snippets updated to current
      pg_tide command, image, environment variable, and stable `relay_set_*_v2`
      SQL API naming
- [x] `pg-tide-relay-fixes.md` research and implementation plan added
- [x] Migration script `sql/pg_ripple--0.126.0--0.127.0.sql`

## Exit Criteria

- Static search finds no current deployment guidance for a pg-trickle relay
- `cargo fmt --check` passes
- `cargo check --workspace` passes
- `scripts/check_dep_versions.sh` passes
- Migration script applies cleanly in the migration-chain test
