# v0.94.0 — Assessment 15 High Remediation

> **Full technical details:** [v0.94.0-full.md](v0.94.0-full.md)

**Status: Released** | **Scope: Large**

> v0.94.0 addresses all Assessment 15 high-severity findings:
> security hardening (search-path injection in SECURITY DEFINER function),
> bump-version recipe improvements, bounded bidi relay channel with overflow
> protection, and the shared `copy_into_vp()` helper enabling the COPY-style
> bulk-load path.

---

## What is this?

This release closes four high-priority issues from Assessment 15:

1. **H15-01** — `just bump-version` recipe now accepts an optional `COMPAT_MIN`
   argument; `check-version-sync` correctly allows `COMPATIBLE_EXTENSION_MIN ≤
   current version` (matching the CI gate).
2. **H15-02** — `_pg_ripple.ddl_guard_vp_tables()` gains `SET search_path = pg_catalog,
   _pg_ripple, public` to prevent search-path injection; a new CI script
   `check_security_definer_search_path.sh` enforces this for all SECURITY
   DEFINER functions in src/.
3. **H15-03/L15-13** — Bounded bidirectional relay channel: new GUC
   `pg_ripple.bidi_relay_max_inflight` (default: 1000) gates relay dispatch;
   overflow increments `pg_ripple_bidi_relay_dropped_total` Prometheus counter.
4. **H15-05/M15-20** — Shared `copy_into_vp()` helper using UNNEST arrays
   extracted to `storage/ops`; new `pg_ripple.bulk_load_use_copy` GUC (default:
   off) enables the COPY-style path for large bulk loads.

## Headline outcomes

### H15-01: bump-version improvements

- [x] **H15-01** — Update `just bump-version NEW_VERSION [COMPAT_MIN]` to accept an optional
  `COMPAT_MIN` parameter that sets `COMPATIBLE_EXTENSION_MIN` independently from the
  release version.  Fix `check-version-sync` recipe to check `COMPAT_MIN ≤ EXT_VER`
  (not `COMPAT_MIN == EXT_VER`), matching the CI `lint-version-sync` gate.
  Bump `COMPATIBLE_EXTENSION_MIN` to v0.93.0 for this release.

### H15-02: SECURITY DEFINER search_path hardening

- [x] **H15-02** — Add `SET search_path = pg_catalog, _pg_ripple, public` to
  `_pg_ripple.ddl_guard_vp_tables()`.  Create
  `scripts/check_security_definer_search_path.sh` that scans `src/**/*.rs` and
  fails if any SECURITY DEFINER function lacks a SET search_path clause.  Wire
  the script into `ci.yml` as the `SECURITY DEFINER search_path lint` step.
  Update `sql/pg_ripple--0.93.0--0.94.0.sql` with the recreated function.

### H15-03: Bounded bidi relay channel

- [x] **H15-03** — Add `pg_ripple.bidi_relay_max_inflight` GUC (default: 1000,
  range 1–100000) registered in `src/gucs/registration/storage.rs`.  In
  `src/bidi/relay.rs`:`ingest_jsonld_impl()`, call `relay_inflight_acquire()`
  before dispatch; if the limit is reached, emit a WARNING and return 0.  Track
  `BIDI_RELAY_INFLIGHT` and `BIDI_RELAY_DROPPED_TOTAL` static `AtomicI64`
  counters in `src/stats.rs`.  Expose both in `streaming_metrics()`.

- [x] **L15-13** — Add `pg_ripple_bidi_relay_dropped_total` Prometheus counter
  to `pg_ripple_http/src/metrics.rs` and `/metrics` endpoint in
  `pg_ripple_http/src/routing/admin_handlers.rs`.

### H15-05: COPY-style bulk load path

- [x] **H15-05** — Add `pg_ripple.bulk_load_use_copy` GUC (default: off).  Extract
  `copy_into_vp()` helper to `src/storage/ops/mod.rs` using UNNEST-array based
  bulk insertion.  In `batch_insert_encoded()`, call `copy_into_vp()` when the
  GUC is on.  Helper is `pub(crate)` for use by R2RML and CDC paths.

- [x] **M15-20** — `copy_into_vp()` helper is the shared insertion point for
  bulk loader, R2RML, and CDC paths, ensuring the COPY-style path is used
  consistently when `bulk_load_use_copy = on`.
