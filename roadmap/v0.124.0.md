## Deliverables checklist

- [x] **(PATH-BNODE-01)** Fix blank-node join in `GraphPattern::Path` translator
  (`src/sparql/sqlgen.rs`): replace `if let TermPattern::Variable` with
  `bgp::bind_term()` for both subject and object so that anonymous blank nodes
  connecting decomposed Sequence paths generate proper `INNER JOIN` conditions
  instead of a Cartesian product.  Affected patterns include `hop*/hop`,
  `hop?/hop`, `^hop/!hop`, and any Sequence whose left operand is ZeroOrMore,
  ZeroOrOne, or an operator spargebra/sparopt decomposes via blank node.

## Notes

- The SPARQL property path algebra implementation was already complete in v0.5.0+;
  this release adds comprehensive test coverage and fixes the PATH-BNODE-01 Cartesian
  product bug that affected Sequence paths decomposed by spargebra/sparopt.
- The `bgp::bind_term()` fix also handles `TermPattern::NamedNode` and
  `TermPattern::Literal` (adding redundant but harmless outer filter conditions when
  those are already baked into the path SQL); this maintains consistency with the
  BGP translator.
- Test 20 (`(hop|^hop)+ from c`) correctly returns 6 results: the starting node `c`
  is itself reachable via `c→d→c` (two bidirectional steps), which is valid SPARQL
  semantics for `p+`.
