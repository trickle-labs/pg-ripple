# v0.110.0 ‚Äî NS-RL Evaluation Harness, Continuous Monitoring & Rule Explainability

**Theme**: Neuro-symbolic record linkage evaluation and operational tooling.

> **In plain language:** Building a record linkage system without a way to measure its accuracy
> is flying blind. This release adds a one-call evaluation function that scores the current
> pipeline against a gold-standard set of verified matches, a CI benchmark gate on two standard
> ER datasets, live monitoring stream tables (always-fresh counts of unresolved entities and
> cluster sizes), and a plain-English rule explanation function so domain experts can review
> AI-generated Datalog rules without needing to read Datalog syntax.
>
> **Foundation document**: [plans/neuro-symbolic-record-linkage.md](../plans/neuro-symbolic-record-linkage.md)
> ¬ß14 (evaluation methodology), ¬ß13.4 (continuous evaluation), ¬ß5.6 (explain_rule),
> ¬ß13.5 (threat model / anomaly detection), synergies 12, 14, 19.
>
> **Effort estimate: 2‚Äì3 person-weeks**

## Deliverables

### Evaluation Harness

- [x] `pg_ripple.evaluate_resolution(gold_graph TEXT, pipeline_options JSONB DEFAULT '{}') ‚Üí JSONB`
  - Runs `resolve_entities()` (or reads from an existing result graph named in `pipeline_options`)
  - Computes and returns all three metric axes (¬ß14.2 of NS-RL plan):
    - **Pairwise**: `precision`, `recall`, `f1`
    - **Blocking**: `pairs_completeness`, `reduction_ratio`, `f_pq`
    - **Cluster (B¬≥)**: `b3_precision`, `b3_recall`, `b3_f1`
  - Result JSONB also includes: `total_gold_pairs`, `total_predicted_pairs`, `true_positives`, `false_positives`, `false_negatives`, `evaluated_at`
  - `gold_graph` must be a named graph containing `owl:sameAs` triples representing verified matches; raises PT0461 if the graph is empty or does not exist
- [x] PT0461 entry in error catalog (`evaluate_resolution: gold graph '%s' is empty or does not exist`)

### Magellan ER Benchmark CI Gate

- [x] `benchmarks/er_magellan.sh` ‚Äî downloads or uses cached Abt-Buy and DBLP-ACM Magellan datasets (cached under `target/benchmarks/magellan/` to avoid repeated downloads in CI; cache key is the dataset checksum), converts to RDF via a bundled Python helper `scripts/magellan_to_rdf.py`, loads into a fresh pg_ripple instance, runs `resolve_entities()`, calls `evaluate_resolution()` against bundled ground-truth named graphs, fails if:
  - Abt-Buy F1 < 0.78 (Splink baseline)
  - DBLP-ACM F1 < 0.90 (Ditto baseline: 0.95 ‚Äî informational target, hard gate at 0.90)
- [x] `benchmarks/er_freshness.sh` ‚Äî inserts 1,000 entity records at 100 rec/s via `pg_ripple.insert_triple()`, asserts symbolic-match (SHACL IMMEDIATE stream table) p95 latency < 500 ms, fails CI on regression
- [x] Both benchmarks added to the `just bench-er` recipe

### Live Monitoring Stream Tables

- [x] `pg_ripple.enable_er_monitoring() ‚Üí VOID` ‚Äî idempotent function that creates:
  - `_pg_ripple.er_unresolved_entities`: compiled from the SPARQL view in synergy 14 (¬ß9 of NS-RL plan), 5 s schedule, counts entities with no `owl:sameAs` link
  - `_pg_ripple.er_cluster_sizes`: union-find statistics stream table (RECURSIVE CTE, 30 s schedule), columns: `canon BIGINT, cluster_size BIGINT`
  - `_pg_ripple.er_resolution_dashboard`: aggregate counts, 10 s schedule: `pending_candidates`, `blocked_merges`, `total_sameas_links`, `entity_clusters`, `largest_cluster`
- [x] `pg_ripple.disable_er_monitoring() ‚Üí VOID` ‚Äî drops the three stream tables if they exist
- [x] pg_regress test: `enable_er_monitoring()` creates all three tables; `disable_er_monitoring()` removes them; dashboard query returns correct counts after a known set of `owl:sameAs` insertions

### `explain_rule()` Plain-English Rule Narration

- [x] `pg_ripple.explain_rule(rule_id BIGINT, language TEXT DEFAULT 'en', format TEXT DEFAULT 'text') ‚Üí TEXT`
  - Retrieves the Datalog rule from `_pg_ripple.rules` by `rule_id`; raises PT0462 if not found
  - Calls the LLM endpoint (`pg_ripple.llm_endpoint`) with a structured system prompt: "Explain in plain English what this Datalog rule does. The audience is a domain expert who does not know Datalog. Rule: ‚Ä¶"
  - Caches result in `_pg_ripple.rule_explanations (rule_id BIGINT, language TEXT, format TEXT, explanation TEXT, generated_at TIMESTAMPTZ)` with invalidation on `pg_ripple.rule_explanation_cache_ttl` GUC (default: `'24 hours'`)
  - **LLM unavailability fallback**: returns a template-driven structural description: "Rule '<name>' derives <head_predicate>(<head_vars>) when all of: <body_atom_1>, <body_atom_2>, ‚Ä¶"
  - `format TEXT`: `'text'` (plain prose) or `'markdown'` (with rule syntax fenced block + prose section)
- [x] `pg_ripple.explain_rule_batch(rule_ids BIGINT[]) ‚Üí TABLE(rule_id BIGINT, explanation TEXT)` ‚Äî batch variant
- [x] REST endpoint `GET /rules/{id}/explain?language=en&format=text` in `pg_ripple_http`
- [x] PT0462 entry in error catalog (`explain_rule: rule %d not found`)
- [x] pg_regress test: `explain_rule()` with a mock LLM endpoint returns non-empty text; fallback path triggered when endpoint URL is empty

### Anomaly Detection for `owl:sameAs`

- [x] `_pg_ripple.sameas_anomaly_log (id BIGSERIAL PRIMARY KEY, detected_at TIMESTAMPTZ DEFAULT now(), entity1 BIGINT, entity2 BIGINT, cluster_size_before INT, cluster_size_after INT, trigger TEXT, transaction_xid XID8)` ‚Äî append-only table; `INSERT ONLY` RLS policy applied (`ALTER TABLE ‚Ä¶ FORCE ROW LEVEL SECURITY`)
- [x] `pg_ripple.record_sameas_anomalies` GUC (BOOL, default `on`): when `on`, any `owl:sameAs` assertion that would exceed `pg_ripple.sameas_max_cluster_size` is logged to `sameas_anomaly_log` before PT550 is raised
- [x] `pg_ripple.sameas_anomaly_log_retention` GUC (INTERVAL, default `'90 days'`): rows older than this are pruned by a background worker pass
- [x] pg_regress test: force a PT550-raising merge, assert `sameas_anomaly_log` has one row with correct entity IDs and cluster sizes
