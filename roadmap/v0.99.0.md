# v0.99.0 — DCTERMS, Schema.org & FOAF Vocabulary Bundles

> **Full technical details:** [v0.99.0-full.md](v0.99.0-full.md)

**Status: Released ✅** | **Scope: Medium**

> v0.99.0 delivers native Datalog rule sets and SHACL shape bundles for the three most widely-used vocabularies in open linked data and enterprise knowledge graphs: Dublin Core Terms (DCTERMS), Schema.org, and FOAF. Together with the SKOS bundle from v0.98.0, this completes the "Big 5" vocabularies that cover 90% of real-world KG interoperability requirements. All three are delivered as named bundles using the `load_datalog_bundle()` / `load_shape_bundle()` API introduced in v0.98.0.

---

## What is this?

Ontologists and knowledge-graph engineers rely on a small set of foundational vocabularies for almost every project. After SKOS (v0.98.0), the three most critical missing vocabularies are:

1. **Dublin Core Terms (DCTERMS)**: The near-universal metadata vocabulary — creator, date, language, subject, rights, format. ~95% of public datasets and linked-data endpoints use DCTERMS. Without it, pg_ripple cannot reason over the most common metadata layer in the linked-data commons.

2. **Schema.org**: The most widely deployed vocabulary on the planet (used by Google, Bing, and millions of websites for structured data). Essential for anything searchable: people, organizations, products, places, events, creative works. Reasoning over Schema.org type hierarchies (`schema:Person` → `schema:Thing`, `schema:LocalBusiness` → `schema:Organization`) is a daily task for enterprise KG teams.

3. **FOAF (Friend-of-a-Friend)**: The de-facto standard for person and social-graph data. `foaf:Person`, `foaf:knows`, `foaf:mbox`, `foaf:Organization` are found in almost every public RDF dataset. FOAF's property-chain rules (inverse properties, `foaf:knows` symmetry, person-account links) are short but high-value.

## Headline outcomes

- [x] `pg_ripple.load_datalog_bundle('dcterms')` — property-hierarchy and sub-property inference (11 rules covering `dcterms:hasPart`/`isPartOf` inverses, refinement chains, `dc11:` compatibility aliases)
- [x] `pg_ripple.load_shape_bundle('dcterms-integrity')` — SHACL shape bundle: 8 validators (cardinality constraints on `dcterms:title`, `dcterms:date` XSD datatype check, `dcterms:language` BCP-47 pattern, `dcterms:creator` range check)
- [x] `pg_ripple.load_datalog_bundle('schema')` — Schema.org type-hierarchy closure (15 rules: `schema:subjectOf`/`schema:about` inverses, `schema:worksFor` → `schema:member` chain, `schema:addressLocality` sub-property propagation, `schema:Thing` type assertion for all schema-typed resources)
- [x] `pg_ripple.load_shape_bundle('schema-integrity')` — 6 validators (required `schema:name`, URL format on `schema:url`, numeric range on `schema:price`, `schema:startDate` ≤ `schema:endDate`)
- [x] `pg_ripple.load_datalog_bundle('foaf')` — 8 rules: `foaf:knows` symmetry, `foaf:Person`/`foaf:Agent` type subsumption, `foaf:account`/`foaf:accountFor` inverse, `foaf:mbox_sha1sum` derivation, `owl:inverseOf` for `foaf:made`/`foaf:maker`, dc-to-foaf creator chain
- [x] `pg_ripple.load_shape_bundle('foaf-integrity')` — 5 validators (IRI pattern on `foaf:homepage`, `foaf:mbox` mailto: check, `foaf:name` cardinality)
- [x] `SELECT * FROM pg_ripple.active_datalog_bundles` — shows all three bundles with `bundle_version`
- [x] Cross-vocabulary Datalog rules: `dcterms:creator` → `foaf:maker` bridge (DC-FOAF-01); `schema:author` → `foaf:maker` bridge (SCHEMA-FOAF-01); `dcterms:subject` → `skos:Concept` type assertion when subject is in a SKOS scheme (DC-SKOS-01)
- [x] SQL helper: `pg_ripple.schema_type_ancestors(iri TEXT)` — returns all Schema.org type ancestors via the type hierarchy closure
- [x] SQL helper: `pg_ripple.foaf_persons()` — returns all `foaf:Person` IRIs visible in the current graph with their `foaf:name` label
- [x] pg_regress test suite: 45+ new tests
- [x] Cookbook chapter `docs/src/cookbook/common-vocabularies.md`
- [x] Blog post `blog/dcterms-schema-foaf-bundles.md`
- [x] Change `pg_ripple.rule_graph_scope` GUC default from `'default'` to `'all'` — ontology-level reasoning across all graphs is the more intuitive behavior for most users

- `pg_ripple.control`: `default_version = '0.99.0'`
