## Deliverables checklist

- [x] **(SNAP-SCHEMA-01)** Create `_pg_ripple.snapshot_id_seq` sequence and
  `_pg_ripple.graph_snapshots` table with columns:
  `(snapshot_id BIGINT DEFAULT nextval('snapshot_id_seq') PRIMARY KEY,
  graph_iri TEXT NOT NULL, snapshot_iri TEXT NOT NULL UNIQUE,
  captured_at TIMESTAMPTZ NOT NULL, triple_count BIGINT, expires_at TIMESTAMPTZ)`.

- [x] **(SNAP-FUNC-01)** `pg_ripple.graph_at(graph_iri TEXT, snapshot_time TIMESTAMPTZ) → TEXT`
  — materialises a snapshot from `_pg_ripple.temporal_facts` (facts where
  `valid_from <= snapshot_time AND (valid_to IS NULL OR valid_to > snapshot_time)`)
  and registers/upserts the snapshot in `_pg_ripple.graph_snapshots`.  Returns
  the deterministic `urn:snapshot:{iri_slug}:{timestamp_utc}` IRI.
  Sets `expires_at = snapshot_time + interval 'N days'` where N =
  `pg_ripple.snapshot_retention_days` (0 = NULL / no expiry).
- [x] **(SNAP-FUNC-02)** `pg_ripple.graph_diff(graph_iri TEXT, from_ts TIMESTAMPTZ, to_ts TIMESTAMPTZ) → TABLE(s BIGINT, p BIGINT, o BIGINT, change TEXT)`
  — returns `'added'`/`'removed'` rows using `EXCEPT`-based set difference
  between the two temporal snapshots.  All IDs are dictionary-encoded BIGINTs.
- [x] **(SNAP-FUNC-03)** `pg_ripple.graph_snapshots_count() → BIGINT`
  — returns `COUNT(*) FROM _pg_ripple.graph_snapshots` for Prometheus gauge
  refresh by the HTTP companion.

- [x] **(SNAP-GC-01)** `crate::temporal::prune_expired_snapshots()` — deletes

## Notes

- The `graph_at()` snapshot IRI is deterministic: given the same `(graph_iri,
  snapshot_time)` pair, repeated calls always return the same IRI and the
  `ON CONFLICT DO UPDATE` upsert refreshes `triple_count` and `expires_at`.
- The `graph_diff()` function uses SQL `EXCEPT` set difference to compute the
  delta, which is correct for the snapshot data model where each fact is
  uniquely identified by its `(s, p, o)` triple within a given graph.
- The Turtle serialisation in the HTTP endpoint is intentionally minimal (one
  triple per line, no prefix shortening) to avoid complexity; clients needing
  compressed Turtle should post-process with a Turtle serialiser.
- The N-Quads delta format includes `# added` / `# removed` comment lines
  before each quad to support streaming audit-compliance consumers.
- `graph_diff()` returns dictionary-encoded `BIGINT` IDs; the HTTP endpoint
  decodes them via `pg_ripple.decode()` before serialisation.
