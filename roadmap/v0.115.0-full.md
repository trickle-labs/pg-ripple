# v0.115.0 — A16 Medium: HTTP API Parity and Observability (Full Details)

**Status: Planned** | **Scope: Large**

## Overview

v0.115.0 closes the HTTP API coverage gap created by the rapid feature development
in v0.95.0–v0.111.0. At the time of Assessment 16, nine major subsystems had
full SQL APIs but no HTTP REST endpoints. This release adds them all, together with
Prometheus metric coverage, a security fix for the pagerank handler, and endpoint
authentication hardening.

> **Effort estimate: 4–5 person-weeks**

### M16-02 — HTTP Route Groups

New routes added to `pg_ripple_http/src/routing/`:

#### Temporal Routes (`routing/temporal_handlers.rs`)

```
GET  /temporal/facts?subject=<iri>&predicate=<iri>    → point-in-time query
POST /temporal/mark                                     → mark_temporal()
POST /temporal/point_in_time                           → point_in_time()
```

Request body for `POST /temporal/mark`:
```json
{
  "subject": "<iri>",
  "predicate": "<iri>",
  "object": "<iri-or-literal>",
  "valid_from": "2024-01-01T00:00:00Z",
  "valid_to": "2024-12-31T23:59:59Z"
}
```

#### PPRL Routes (`routing/pprl_handlers.rs`)

```
POST /pprl/bloom_encode     → bloom_encode(value, key, hash_count, filter_size)
POST /pprl/dice_similarity  → dice_similarity(bloom_a, bloom_b)
```

#### Differential Privacy Routes (`routing/dp_handlers.rs`)

```
POST /dp/noisy_count      → dp_noisy_count(query, epsilon)
POST /dp/noisy_histogram  → dp_noisy_histogram(query, epsilon, buckets)
```

All DP endpoints require `check_auth_write` (epsilon expenditure modifies state).

#### Entity Resolution Routes (`routing/er_handlers.rs`)

```
POST   /entity-resolution/resolve             → resolve_entities()
POST   /entity-resolution/evaluate            → evaluate_er()
POST   /entity-resolution/monitoring/enable   → er_monitoring_enable()
POST   /entity-resolution/monitoring/disable  → er_monitoring_disable()
```

#### Proof Tree Routes (`routing/proof_handlers.rs`)

```
GET /proof-tree/{subject}/{predicate}/{object}
  → explain_proof_tree(subject, predicate, object)
  → returns JSON { "subject": "...", "predicate": "...", "object": "...", "tree": [...] }
```

#### Tenant Routes (extended `routing/admin_handlers.rs`)

```
GET    /tenants/{name}   → get tenant quota and metadata
POST   /tenants/{name}   → create_tenant(name)
DELETE /tenants/{name}   → drop_tenant(name, cascade := false)
```

DELETE requires `check_auth_write` and a `?confirm=true` query parameter.

### M16-04 — SQL Injection Fix in `pagerank_handlers.rs`

**Before (vulnerable):**
```rust
let query = format!(
    "SELECT * FROM pg_ripple.pagerank_scores WHERE direction = '{}'",
    params.direction.replace('\'', "''")
);
```

Manual escaping is fragile and incorrect for all injection vectors.

**After (parameterised):**
```rust
#[derive(Deserialize)]
#[serde(rename_all = "lowercase")]
enum Direction { Inbound, Outbound, Both }

let query = "SELECT * FROM pg_ripple.pagerank_scores WHERE direction = $1";
let rows = client.query(query, &[&direction.to_string()]).await?;
```

The `Direction` enum is deserialised before the handler executes, so invalid
values are rejected at the HTTP layer with a 422 error.

**Impact:** Eliminates OWASP A03:2021 (Injection) risk in this handler.

### M16-09 — Liveness vs Readiness Documentation

`pg_ripple_http/README.md` health-check section:

```markdown
## Health Endpoints

| Endpoint  | Type      | When healthy |
|-----------|-----------|-------------|
| GET /health | Liveness | Process is running; always returns 200 |
| GET /ready  | Readiness | Connected to PostgreSQL AND extension schema version matches |
```

`/ready` returns 503 if:
- PostgreSQL connection pool is exhausted
- Extension version is below `COMPATIBLE_EXTENSION_MIN`

Kubernetes probe annotations in `charts/pg_ripple/values.yaml`:
```yaml
livenessProbe:
  httpGet:
    path: /health
    port: 8080
readinessProbe:
  httpGet:
    path: /ready
    port: 8080
```

### M16-22 — Metrics Endpoint Authentication

When `PG_RIPPLE_HTTP_METRICS_TOKEN` environment variable is set:

1. All requests to `GET /metrics` must include `Authorization: Bearer <token>`
2. The comparison uses `subtle::ConstantTimeEq` to prevent timing attacks
3. On mismatch: `401 Unauthorized` with `WWW-Authenticate: Bearer realm="pg_ripple"`
4. Token is read once at startup and stored in `AppState`

Implementation in `pg_ripple_http/src/common.rs`:
```rust
pub fn check_metrics_auth(
    headers: &HeaderMap,
    expected_token: Option<&str>,
) -> Result<(), StatusCode> {
    let Some(expected) = expected_token else { return Ok(()); };
    let bearer = headers.get(AUTHORIZATION)
        .and_then(|v| v.to_str().ok())
        .and_then(|s| s.strip_prefix("Bearer "))
        .unwrap_or("");
    if bearer.as_bytes().ct_eq(expected.as_bytes()).unwrap_u8() == 1 {
        Ok(())
    } else {
        Err(StatusCode::UNAUTHORIZED)
    }
}
```
