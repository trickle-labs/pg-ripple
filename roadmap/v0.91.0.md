# v0.91.0 — Assessment 14 Medium: Observability, API, Standards, Build & Documentation

> **Full technical details:** [v0.91.0-full.md](v0.91.0-full.md)

**Status: ✅ Released** | **Scope: Medium**

> v0.91.0 closes the remaining Medium-severity findings from PLAN_OVERALL_ASSESSMENT_14 that fall outside the correctness/performance/quality scope of v0.90.0. Key areas: Prometheus metrics for PageRank IVM queue depth; SHACL score-log retention policy; SSE endpoint verification; HTTP routing middleware extraction; Arrow Flight COUNT(*) estimation swap; GUC API documentation cleanup; PT error code registry completeness; SPARQL 1.2 and RDF-star compliance matrix updates; lint-version-sync and migration-chain CI jobs; dependency upgrades (arrow, parquet); IVM boundary documentation; CDC LSN watermark batching; and compatibility matrix rows for v0.87/v0.88.

---

## What is this?

After v0.89.0 (High findings) and v0.90.0 (Medium correctness/performance/code-quality), this release closes the remaining ~25 Medium findings that span observability, API design, standards compliance, build system, supply chain, IVM, CDC, and documentation. These are individually small (mostly S or M effort) but collectively ensure the codebase is production-ready from an operations and documentation perspective before the v1.0.0 API freeze.

---

## Headline outcomes

### Observability

- [x] **OBS-01** — `pg_ripple_http/src/metrics.rs` extended with three PageRank IVM Prometheus gauges: `pg_ripple_pagerank_queue_depth` (dirty edges queued), `pg_ripple_pagerank_queue_max_delta` (largest accumulated score delta), `pg_ripple_pagerank_queue_oldest_enqueue_seconds` (age of oldest queued edge in seconds). Polled from `pg_ripple.pagerank_queue_stats()` by the metrics scrape handler. Labels: `{topic}`.
- [x] **OBS-02** — `pg_ripple.shacl_score_log_retention_days` GUC (INT, default 30). Background vacuum worker step added to `_PG_init` to run daily: `DELETE FROM _pg_ripple.shacl_score_log WHERE logged_at < NOW() - INTERVAL '? days'`. Prevents unbounded table growth.
- [x] **OBS-03** — `docs/src/operations/observability.md` extended: documents that `pgrx::log!` calls emit PostgreSQL-structured log messages, and when `log_destination = jsonlog` is active, fields are mapped to PostgreSQL's JSON log format without duplication. No changes to extension source required.

### HTTP Companion

- [x] **HTTP-02** — `pg_ripple_http/src/stream.rs` (SSE path) verified to be wired in `pg_ripple_http/src/routing/mod.rs` for the `/sparql/stream` endpoint. Regression test added: subscribes via SSE to a live endpoint, asserts at least one `data:` event is received within the test timeout.
- [x] **HTTP-03** — CORS / rate-limiting / tracing middleware composition extracted from `pg_ripple_http/src/main.rs` to `pg_ripple_http/src/routing/middleware.rs`. `build_router()` in `routing/mod.rs` imports from `middleware.rs`. `main.rs` length reduced from ~250 to ~150 lines.
- [x] **HTTP-04** — `pg_ripple_http/src/arrow_encode.rs` pre-check replaces `COUNT(*)` with `EXPLAIN (FORMAT JSON, ANALYZE false) SELECT ... LIMIT 1` to extract the planner's row estimate. For large queries the count was itself a full scan; the planner estimate is available immediately. Falls back to COUNT(*) if estimate is unavailable (e.g. statistics not yet collected).

### API Design

- [x] **API-02** — `pg_ripple.shacl_report_scored()` column order pinned by regression test in `tests/pg_regress/sql/shacl.sql`. Documented in `docs/src/reference/sql-functions.md`.
- [x] **API-03** — `pg_ripple.pagerank_run()` all parameters documented with named-argument examples in `docs/src/features/pagerank.md`. Example showing `SELECT * FROM pg_ripple.pagerank_run(damping => 0.85, max_iterations => 50, topic => 'citation')`.
- [x] **API-04** — `pg:confidence(?s, ?p, ?o)` and `pg:pagerank(?node, ?topic)` function IRIs pinned to `http://pg-ripple.org/fn/` namespace: `http://pg-ripple.org/fn/confidence`, `http://pg-ripple.org/fn/pagerank`. Documented in `docs/src/reference/sparql-extension-functions.md`. The `pg:` prefix prefix-declaration in the SPARQL parser maps to this namespace.
- [x] **API-05** — `pg_ripple.explain_pagerank_json(node_iri TEXT, top_k INT DEFAULT 5) RETURNS JSONB` added as a companion to `explain_pagerank()` (which returns a table). Useful for SREs scripting the output.
- [x] **API-06** — `docs/src/reference/error-codes.md` extended with PT0301–PT0307 (v0.87.0 confidence/fuzzy) and PT0401–PT0423 (v0.88.0 PageRank). Verified by `grep -E "PT0[34][0-9]{2}" docs/src/reference/error-codes.md`.

### Standards Conformance

- [x] **STD-01** — `plans/sparql12_tracking.md` updated against W3C SPARQL 1.2 Working Draft (April 2026 snapshot). Each pg_ripple-implemented feature marked with the version of first appearance.
- [x] **STD-02** — `docs/src/reference/sparql-compliance.md` extended with RDF-star SPARQL-star support matrix: `<<>>` in `BIND`, `FILTER`, `CONSTRUCT`, `SELECT` positions verified against the RDF 1.2 (RDF-star) draft. Gaps documented.
- [x] **STD-03** — `docs/src/features/uncertain-knowledge.md` — noisy-OR composition section marked as pg_ripple extension, not a W3C standard. Citation added: De Raedt, Kimmig & Toivonen (2007), *ProbLog: A Probabilistic Prolog*.

### Build System & CI

- [x] **BUILD-01** — `lint-version-sync` CI job in `.github/workflows/ci.yml` (already specified in v0.89.0 ROAD-02). This item verifies it is present and extends it to also check `pg_ripple_http/Cargo.toml` version parity.
- [x] **BUILD-02** — `.github/workflows/migration-chain.yml` dedicated workflow created. Previously the migration chain test was called from `ci.yml` but had no dedicated workflow for PR-comment visibility. Now runs on every PR and produces a distinct check named "Migration chain test".

### Dependencies

- [x] **DEP-01** — `ureq` upgrade triage completed. v0.87/v0.88 did not migrate from `ureq = "2"` to `ureq = "3"`. If ureq 3.x is stable and available, upgrade `pg_ripple_http/Cargo.toml`; document HTTP/2 federation availability. Defer to post-v1.0.0 if API churn is significant.
- [x] **DEP-02** — `arrow` and `parquet` crate versions triaged: `cargo update -p arrow -p parquet --dry-run`. Bump if newer minor versions are available. Update `pg_ripple_http/Cargo.toml`.

### IVM

- [x] **IVM-01** — `docs/src/architecture/ivm.md` new section "IVM Boundary: CWB vs PageRank": explains that CWB-IVM (CONSTRUCT writeback via Delete-Rederive) and PageRank-IVM (dirty-edge queue + K-hop propagation) are independent mechanisms that do not interact. Documents the maintenance boundary.
- [x] **IVM-02** — Regression test added to `tests/pg_regress/sql/construct_rules.sql`: verifies `run_full_recompute` propagates confidence on full re-computation when `pg_ripple.prov_confidence = on`. Specifically tests the `CWB confidence propagation flag` path.

### CDC & Streaming

- [x] **CDC-01** — `src/cdc.rs` audited for per-event vs batched LSN watermark updates. If batching (CC13-03) did not land in v0.85.0, implement: accumulate watermarks in a local buffer, flush every `pg_ripple.cdc_watermark_batch_size` events (INT GUC, default 100) or every `pg_ripple.cdc_watermark_flush_interval_ms` ms (INT GUC, default 50).
- [x] **CDC-02** — `benchmarks/bidi_relay_throughput.sql` CI scaling test wired to `performance_trend.yml`; asserts throughput within ±20% of baseline.

### Documentation

- [x] **DOC-01** — `docs/src/operations/compatibility.md` — rows for v0.87.0 and v0.88.0 appended to the extension/HTTP companion compatibility matrix.
- [x] **DOC-02** — `docs/src/features/pagerank.md` completeness audit: all 22 v0.88.0 GUCs and 5 SQL functions (`pagerank_run`, `explain_pagerank`, `pagerank_queue_stats`, `centrality_run`, `export_pagerank`) documented with at least one example each.
