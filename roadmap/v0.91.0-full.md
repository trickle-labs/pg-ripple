async fn estimate_row_count(query: &str, client: &SpiClient) -> Option<i64> {
    let explain_query = format!("EXPLAIN (FORMAT JSON, ANALYZE FALSE) {}", query);

    let plan_rows: i64 = extract_plan_rows(&result)?;

let count = estimate_row_count(query, &client).await

   `<http://pg-ripple.org/fn/confidence>(?s, ?p, ?o)`

## 7. IVM

### IVM-01 — IVM Boundary Documentation

`docs/src/architecture/ivm.md` new section:

> ## IVM Mechanism Boundary
>
> pg_ripple implements two independent incremental maintenance mechanisms:
>
> **CWB-IVM** (CONSTRUCT Writeback IVM): maintains inferred triples derived from SPARQL CONSTRUCT rules. Uses the Delete-Rederive algorithm with Z-set deltas. Source: `src/construct_rules/delta.rs`. Triggered by VP table delta inserts via `_pg_ripple.cwb_queue`.
>
> **PageRank-IVM** (K-hop local push): maintains approximate PageRank scores. Uses bounded K-hop propagation from dirty edges. Source: `src/pagerank/ivm.rs`. Triggered by VP table delta inserts via `_pg_ripple.pagerank_dirty_edges`.
>
> The two mechanisms are **independent**: CWB-IVM does not trigger PageRank recomputation, and PageRank-IVM does not write inferred triples back to VP tables. A full `pagerank_run()` is required after a large CWB recompute if edge predicates are affected.
