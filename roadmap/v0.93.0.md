# v0.93.0 — pg_tide Integration & Documentation Modernisation

> **Full technical details:** [v0.93.0-full.md](v0.93.0-full.md) (reference will be created as full details emerge)

**Status: Released** | **Scope: Small**

> v0.93.0 integrates the newly extracted `pg_tide` extension (which now contains all relay, outbox, and inbox subsystem logic previously in pg_trickle) and modernises pg_ripple's documentation to reflect the new architecture. After v0.46.0, pg_trickle provides IVM only; all bidirectional relay functionality moved to the standalone `pg_tide` extension. See [plans/PLAN_PG_TIDE.md](../plans/PLAN_PG_TIDE.md) for the full impact analysis.

---

## What is this?

The pg-trickle project extracted its relay, outbox, and inbox subsystem into a standalone `pg_tide` extension (`trickle-labs/pg-tide`). This release updates pg_ripple to acknowledge pg_tide as the recommended relay transport layer while maintaining backward compatibility with pg_trickle for IVM.
