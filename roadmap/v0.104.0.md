# v0.104.0 — Domain Rule Library Infrastructure

**Theme**: Package, share, and install domain-specific Datalog rule sets as versioned libraries.

> **In plain language:** Right now, every pg_ripple user has to write their own reasoning rules
> from scratch. A hospital team writes their own patient-matching rules. A bank team writes their
> own AML (anti-money laundering) rules. A logistics company writes their own supply-chain rules.
> None of that work is shared. This release creates the infrastructure for distributing these
> rule sets as reusable libraries — like npm packages or pip packages, but for Datalog rules and
> SHACL shapes. A library is a Turtle file with metadata, installed with a single SQL function
> call. Dependencies are resolved automatically. Users can build on community-maintained rule
> sets rather than starting from zero.
>
> **Foundation document**: [plans/expert-system.md](../plans/expert-system.md) §5
>
> **Effort estimate: 2–3 person-weeks**

## Prerequisites

- v0.99.0 released and all tests passing (named bundle API via `load_datalog_bundle()` / `load_shape_bundle()` available).
- Independent of v0.100.0–v0.103.0 — can ship in parallel.

### Relationship to Built-in Bundles

The built-in bundle system (v0.98.0–v0.99.0: `load_datalog_bundle('skos')`, etc.) is hard-coded in Rust and ships with pg_ripple. Rule libraries (this version) are an **external, user-extensible** counterpart: libraries are Turtle files loaded from URLs or local paths and installed into a catalog table. The two systems coexist:
- Built-in bundles remain available via `load_datalog_bundle()` as before.
- External rule libraries use `install_rule_library()` and are stored in `_pg_ripple.rule_libraries`.
- A rule library may declare a built-in bundle as a dependency (e.g. `pg:dependsOn "skos"`) — `install_rule_library` activates the bundle via `load_datalog_bundle()` before loading the library's own rules.
- Naming collision (a library with the same name as a built-in bundle) is rejected with PT0459.

## Deliverables

### Rule Library Format Specification

A rule library is a Turtle file containing:
- [ ] One or more Datalog rules in the `_pg_ripple` rule syntax (embedded as `xsd:string` literals on a `pg:ruleLibrary` subject)
- [ ] Zero or more SHACL shapes (as named shapes in the graph)
- [ ] Required metadata triples:
  - `dcterms:title "My Library"` — human-readable name
  - `dcterms:description "..."` — what the library does
  - `dcterms:license <https://spdx.org/licenses/MIT.html>` — SPDX license IRI
  - `owl:versionInfo "1.0.0"` — version string
  - `pg:dependsOn <url> ;` — zero or more dependency library URLs (acyclic, single-version)
- [ ] A documentation chapter `docs/src/cookbook/rule-libraries.md` explaining the format, how to author and publish a library, and what license and disclaimer requirements operators should evaluate

### `_pg_ripple.rule_libraries` Catalog Table

- [ ] `_pg_ripple.rule_libraries (name TEXT PRIMARY KEY, version TEXT NOT NULL, installed_at TIMESTAMPTZ DEFAULT now(), description TEXT, license_iri TEXT, source_url TEXT, dependencies TEXT[])` — one row per installed library

### `install_rule_library()` SQL Function

- [ ] `pg_ripple.install_rule_library(source TEXT, accept_license BOOLEAN DEFAULT FALSE) → TEXT`
  - `source` is either a URL or an absolute local file path
  - URL sources are validated against the existing SSRF allowlist (`pg_ripple.federation_ssrf_allowlist`) before any network request is made — raises PT0452 if blocked
  - Parses the Turtle file, extracts library metadata, rules, and SHACL shapes
  - **Dependency resolution**: fetches and installs declared `pg:dependsOn` libraries first, in topological order; raises PT0453 on dependency cycles; raises PT0454 on missing (unreachable) dependencies
  - Raises PT0455 when `accept_license = FALSE` and the library's license IRI is not in `{'https://spdx.org/licenses/MIT.html', 'https://spdx.org/licenses/Apache-2.0.html', 'https://spdx.org/licenses/PostgreSQL.html'}` (i.e. any non-permissive or unknown license requires explicit acceptance)
  - Returns the library name on success
  - Idempotent: re-installing the same version is a no-op and returns the name without error

### `upgrade_rule_library()` and `uninstall_rule_library()`

- [ ] `pg_ripple.upgrade_rule_library(name TEXT) → TEXT` — re-fetches from `source_url`, replaces rules and shapes, updates version; raises PT0456 if another installed library depends on `name` and the new version would break the dependency contract
- [ ] `pg_ripple.uninstall_rule_library(name TEXT) → VOID` — removes rules and shapes installed by this library; raises PT0456 if another installed library depends on `name`
- [ ] `pg_ripple.list_rule_libraries() → TABLE(name TEXT, version TEXT, installed_at TIMESTAMPTZ, description TEXT, license_iri TEXT)` — lists all installed libraries

### Error Catalog

- [ ] PT0452: `install_rule_library: URL '%s' is blocked by the SSRF allowlist`
- [ ] PT0453: `install_rule_library: dependency cycle detected involving library '%s'`
- [ ] PT0454: `install_rule_library: dependency '%s' could not be fetched`
- [ ] PT0455: `install_rule_library: library '%s' uses license '%s' — set accept_license => TRUE to confirm acceptance`
- [ ] PT0456: `%s_rule_library: library '%s' is required by '%s' — uninstall the dependent first`
- [ ] PT0459: `install_rule_library: name '%s' conflicts with a built-in bundle`

### REST Endpoint

- [ ] `GET /rule-libraries` in `pg_ripple_http` — returns the `list_rule_libraries()` result as JSON

### Tests
