# v0.90.0 — Assessment 14 Medium: Correctness, Performance, Concurrency & Code Quality

> **Full technical details:** [v0.90.0-full.md](v0.90.0-full.md)

**Status: Released ✅** | **Scope: Large**

> v0.90.0 sweeps all Medium-severity findings from PLAN_OVERALL_ASSESSMENT_14 across six dimensions: correctness (PageRank convergence norm documentation, incremental K-hop drift bounds, SPARQL MINUS blank-node scope, export format validation), performance (WCOJ integration for large-graph PageRank, temp-materialisation avoidance, embedding fast-path GUC gate, confidence ANALYZE after bulk load, `clippy::unwrap_used` workspace lint), concurrency (`pagerank_dirty_edges` deadlock test, confidence hot-row benchmark, advisory lock for concurrent PageRank runs), test coverage (confidence + PageRank proptests, confidence-loader fuzz target, PageRank scale gate, concurrent SPARQL+PageRank test), code quality (pre-emptive splits of seven 1,300–1,700 line files, `src/pagerank.rs` module split, `src/uncertain_knowledge_api.rs` module split, `pg_ripple_http/src/datalog.rs` routing migration), and Datalog/reasoning (probabilistic weight parser validation, cyclic convergence documentation).

---

## What is this?

Assessment 14 identified 51 Medium findings. v0.89.0 consumed 5 of those Medium findings (security escalations). v0.90.0 closes the remaining 46 Medium findings in a single "quality sweep" release that mirrors the v0.85.0/v0.86.0 pattern from A13 remediation.

The highest-effort item here is **PERF-01**: integrating the existing WCOJ executor (`src/sparql/wcoj.rs`, delivered in v0.79.0) with the PageRank per-iteration neighbour scan. For graphs exceeding 10M edges the SQL planner's per-iteration hash join serialises through the planner for each of up to 100 iterations; routing through WCOJ when `pg_ripple.wcoj_enabled = on` and `edge_count > 10M` yields an estimated 5–20× improvement for dense power-law graphs.

The module splits (**CQ-02, CQ-03, CQ-04, CQ-05**) are the second-largest effort. Seven files are in the 1,300–1,700 line range, all approaching the 1,800-line CI gate established in v0.85.0. Pre-emptive splits prevent the gate from failing mid-feature and maintain single-responsibility structure.

## Headline outcomes

### Correctness

- [x] **CB-02** — `docs/src/features/pagerank.md` documents that L1 norm is used for convergence (consistent with NetworkX); `pg_ripple.pagerank_convergence_norm` GUC added (`'l1'` | `'l2'` | `'linf'`, default `'l1'`) to support alternative norms.
- [x] **CB-04** — PageRank IVM K-hop drift bound documented: `|Δscore| ≤ alpha^k * max_delta_per_iteration`. `pg_ripple.pagerank_full_recompute_threshold` GUC (FLOAT8, default 0.01) triggers a scheduled full recompute when the accumulated dirty fraction exceeds the threshold. Documented in `docs/src/features/pagerank.md`.
- [x] **CB-05** — `tests/pg_regress/sql/sparql_minus_blank_scope.sql` added: SPARQL `MINUS` with shared blank-node scope regression test mirroring the A13 OPTIONAL+EXISTS test.
- [x] **CB-06** — `pg_ripple.export_pagerank()` raises `PT0417` (was silently defaulting to CSV) for any `format` value not in `{'turtle','jsonld','csv','ntriples'}`. Error message: `"PT0417 unsupported export format '%s' — valid values: turtle, jsonld, csv, ntriples"`.

### Performance

- [x] **PERF-01** — PageRank per-iteration neighbour scan wired through WCOJ executor when `pg_ripple.wcoj_enabled = on` AND estimated edge count > 10M (from `_pg_ripple.predicates.triple_count`). The WCOJ path replaces the nested-loop join with a Leapfrog Triejoin over the sorted VP-table arrays. `explain_inference()` reports `pagerank_scan_mode: 'wcoj' | 'sql'`. Guarded by `pg_ripple.pagerank_wcoj_threshold` GUC (BIGINT, default 10000000).
- [x] **PERF-02** — Sketch top-K documented: sketch type is Count-Min Sketch (CMS); `pg_ripple.pagerank_sketch_width` GUC (INT, default 2000) and `pg_ripple.pagerank_sketch_depth` GUC (INT, default 5). Memory bound: `width × depth × 8` bytes (documented in `docs/src/features/pagerank.md`). Accuracy guarantee: `ε = e/width` error probability `δ = e^{-depth}`.
- [x] **PERF-03** — `clippy::unwrap_used` and `clippy::expect_used` added to workspace lint config in `Cargo.toml`. All existing `.unwrap()` / `.expect(` calls in v0.87/v0.88 modules audited; each receives either a `// CLIPPY-OK: <reason>` comment or conversion to `pgrx::error!()` / `?`-propagation. CI blocks on new violations.
- [x] **PERF-04** — `pg_ripple.pagerank_run()` avoids temp-table materialisation when `estimated_edge_count * 8 bytes < work_mem * pagerank_partitions`; streams directly from VP tables. The materialisation path remains as a fallback for edge counts above the threshold. New GUC `pg_ripple.pagerank_temp_threshold` (BIGINT, default `work_mem / 8` bytes equivalent in edges) controls the cutoff.
- [x] **PERF-05** — `src/sparql/embedding.rs` entry point gates on `pg_ripple.pgvector_enabled` GUC (bool); when off, all embedding dispatch returns `PT0501 pgvector extension not enabled` immediately without loading the module. Reduces overhead for deployments without pgvector.
- [x] **PERF-06** — `load_triples_with_confidence()` runs `ANALYZE _pg_ripple.confidence` at completion via `SpiClient::execute`. Confidence join estimates now use real statistics.

### Concurrency

- [x] **CON-01** — `tests/concurrency/pagerank_during_merge.sh` pgbench-driven test: concurrent SPARQL + PageRank + HTAP merge; asserts no deadlock, no data corruption. Covers the `pagerank_dirty_edges` primary-key index concurrent-insert scenario.
- [x] **CON-02** — `benchmarks/probabilistic_overhead.sql` extended with a pgbench scenario that stress-tests concurrent `INSERT ON CONFLICT DO UPDATE SET confidence = noisy_or(...)` on `_pg_ripple.confidence`. Baseline added to `benchmarks/merge_throughput_baselines.json`.
- [x] **CON-03** — `pg_ripple.pagerank_run()` acquires `pg_advisory_xact_lock(hashtext('pagerank_run_' || coalesce(topic, '')))` at function entry. Two concurrent calls to the same topic block rather than racing on `TRUNCATE + INSERT` of `pagerank_scores`. Locking semantics documented in `docs/src/features/pagerank.md`.

### Test Coverage

- [x] **TEST-02** — `tests/proptest/pagerank_oracle.rs` proptest: builds random directed graphs (5–100 nodes, 10–500 edges), computes PageRank via a Rust reference implementation (power iteration), compares against `pg_ripple.pagerank_run()` output, asserts max difference < 1e-4. Uses `tests/proptest/ntriples_oxigraph.rs` harness shape. Complements CB-01's confidence algebra proptest.
- [x] **TEST-03** — `fuzz/fuzz_targets/confidence_loader.rs`: fuzzes `load_triples_with_confidence()` input; asserts NaN, ±Inf, denormal floats, and out-of-range [0,1] values raise `PT0302` and are not stored.
- [x] **TEST-04** — `benchmarks/pagerank_scale.sh`: 1M / 10M edge synthetic scale-free graphs; asserts convergence within 100 iterations and wall time within ±20% of baseline. Wired to `performance_trend.yml`.
- [x] **TEST-05** — `tests/concurrency/pagerank_during_merge.sh` (see CON-01). Additionally `tests/concurrency/confidence_subxact_rollback.sql` verifies the noisy-OR aggregation rows are rolled back with their sub-transaction (CON-05 / Low).

### Code Quality — Module Splits

- [x] **CQ-02** — Pre-emptive splits of the seven 1,300–1,700 line files approaching the 1,800-line CI gate. Priority order: `src/sparql/execute.rs` (1,470 lines) → split into `execute/{core,construct,describe,update,explain}.rs`; `src/sparql/expr.rs` (1,610) → split into `expr/{functions,filters,aggregates,cast}.rs`; `src/datalog/compiler.rs` (1,613) → see CQ-02/DL-03 below.
- [x] **CQ-03** — `src/pagerank.rs` (1,015 lines) split into `src/pagerank/{executor.rs, ivm.rs, sketch.rs, centrality.rs, export.rs, explain.rs, mod.rs}`. Follows the v0.69.0 (`construct_rules/`) and v0.85.0 (`schema/`, `federation/`) split pattern.
- [x] **CQ-04** — `src/uncertain_knowledge_api.rs` split into `src/uncertain/{api.rs, fuzzy.rs, shacl.rs, prov.rs, confidence_table.rs, mod.rs}`. Enables independent extension of fuzzy SPARQL without touching confidence-table internals.
- [x] **CQ-05** — `pg_ripple_http/src/datalog.rs` (1,232 lines) moved into `pg_ripple_http/src/routing/datalog_handlers.rs`; helpers extracted to `routing/datalog_helpers.rs`. Brings it into the established routing split pattern.

### Datalog & Reasoning

- [x] **DL-01** — `src/datalog/parser.rs` rejects malformed `@weight` annotations: `@weight(NaN)` → `PT0301`; `@weight(-0.5)` → `PT0301` ("weight must be in [0, 1]"); `@weight(2.0)` → `PT0301`. Proptest reject-case suite added.
- [x] **DL-02** — `docs/src/features/uncertain-knowledge.md` extended with a "Convergence guarantees for cyclic probabilistic rules" section: noisy-OR is monotone on [0,1]; iteration with `prob_datalog_max_iterations` and `prob_datalog_convergence_delta` converges when the underlying logic program has a well-founded model with respect to noisy-OR; cites the ProbLog fixed-point theorem (De Raedt et al. 2007).
- [x] **DL-03** — `src/datalog/compiler.rs` (1,613 lines) pre-emptive split: `src/datalog/compiler/{sql.rs, prob.rs, shacl_rules.rs, builtins.rs, mod.rs}`. Included in CQ-02 batch.
