# v0.120.0 — Operational & Ecosystem Features: explain_pagerank HTTP, Diagnostic Snapshot, Rule-Library Federation, Read-Replica Routing

**Status:** ✅ Released

## Summary

v0.120.0 closes the Platform Maturity phase with four operational and ecosystem
features: `GET /pagerank/explain/{node}` HTTP endpoint (Feature 7), a diagnostic
snapshot endpoint for support tickets (Feature 8), rule-library federation between
pg_ripple instances via Arrow Flight (Feature 11), and read-replica routing in
`pg_ripple_http` for SPARQL SELECT/CONSTRUCT/ASK (Feature 12). Also adds tenant
quota HTTP endpoints, Helm PodDisruptionBudget, and a `just generate-helm-values`
recipe (Feature 9).

## Deliverables

### Feature 7 — `GET /pagerank/explain/{node_iri}` HTTP Endpoint

- [x] Implement `GET /pagerank/explain/{node_iri}` in `routing/pagerank_handlers.rs`
- [x] Wrap existing `explain_pagerank()` SQL function
- [x] Return JSON: `{"node": "...", "score": 0.xx, "top_contributors": [{...}], "method": "..."}`
- [x] URL-encode IRI in path; handler decodes with `urlencoding::decode()`
- [x] Authentication: read-only (requires `check_auth` but not `check_auth_write`)

### Feature 8 — `GET /admin/diagnostic-snapshot` Endpoint

- [x] Implement `GET /admin/diagnostic-snapshot` in `routing/admin_handlers.rs`
- [x] Requires `check_auth_write` (sensitive information)
- [x] Returns single JSON archive containing:
  - `_pg_ripple.*` table row counts
  - All non-sensitive GUC values (exclude `pg_ripple.llm_api_key` etc.)
  - 60-second Prometheus metrics snapshot
  - Extension version and HTTP companion version
- [x] Suitable for attaching to support tickets without manual SQL

### Tenant Quota HTTP Endpoints

- [x] Implement `GET /tenants/{name}/quota` wrapping `_pg_ripple.quota_*` trigger data
- [x] Implement `POST /tenants/{name}/quota` (create/update quota)
- [x] Write endpoints require `check_auth_write`

### Helm PodDisruptionBudget

- [x] Add `podDisruptionBudget` stanza to `charts/pg_ripple/values.yaml`:
  ```yaml
  podDisruptionBudget:
    enabled: true
    minAvailable: 1
  ```
- [x] Add corresponding PodDisruptionBudget template to `charts/pg_ripple/templates/`
- [x] Document in `charts/pg_ripple/README.md`

### Feature 11 — Rule-Library Federation

- [x] Implement `pg_ripple.publish_rule_library(name TEXT, endpoint_uri TEXT)` SQL function
- [x] Implement `pg_ripple.subscribe_rule_library(source_uri TEXT, name TEXT)` SQL function
- [x] Publish endpoint: `GET /rule-libraries/{name}/stream` (Arrow Flight stream)
- [x] Subscribe: fetch from remote Flight endpoint and install locally
- [x] Authentication: reuse existing Arrow Flight HMAC ticket mechanism
- [x] Document in `docs/src/cookbook/rule-federation.md`

### Feature 12 — Read-Replica Routing

- [x] Honour `?replica=ok` query parameter on read-only SPARQL SELECT/CONSTRUCT/ASK
- [x] Proxy to read-replica DSN configured in `PG_RIPPLE_HTTP_REPLICA_DSN` env when set
- [x] Log routing decision at DEBUG level
- [x] Fall back to primary if replica is unavailable
- [x] Document in `pg_ripple_http/README.md`

### Feature 9 — `just generate-helm-values` Recipe

- [x] Add `generate-helm-values TENANT=<name>` recipe to `justfile`
- [x] Query `_pg_ripple.tenants` and emit per-tenant `values-<name>.yaml` fragment
- [x] Output suitable for `helm install --values values-<name>.yaml`
- [x] Document in `charts/pg_ripple/README.md`
