# v0.120.0 — Operational & Ecosystem Features: explain_pagerank HTTP, Diagnostic Snapshot, Rule-Library Federation, Read-Replica Routing

**Status:** 🔜 Planned

## Summary

v0.120.0 closes the Platform Maturity phase with four operational and ecosystem
features: `GET /pagerank/explain/{node}` HTTP endpoint (Feature 7), a diagnostic
snapshot endpoint for support tickets (Feature 8), rule-library federation between
pg_ripple instances via Arrow Flight (Feature 11), and read-replica routing in
`pg_ripple_http` for SPARQL SELECT/CONSTRUCT/ASK (Feature 12). Also adds tenant
quota HTTP endpoints, Helm PodDisruptionBudget, and a `just generate-helm-values`
recipe (Feature 9).

## Deliverables

### Feature 7 — `GET /pagerank/explain/{node_iri}` HTTP Endpoint

- [ ] Implement `GET /pagerank/explain/{node_iri}` in `routing/pagerank_handlers.rs`
- [ ] Wrap existing `explain_pagerank()` SQL function
- [ ] Return JSON: `{"node": "...", "score": 0.xx, "top_contributors": [{...}], "method": "..."}`
- [ ] URL-encode IRI in path; handler decodes with `urlencoding::decode()`
- [ ] Authentication: read-only (requires `check_auth` but not `check_auth_write`)

### Feature 8 — `GET /admin/diagnostic-snapshot` Endpoint

- [ ] Implement `GET /admin/diagnostic-snapshot` in `routing/admin_handlers.rs`
- [ ] Requires `check_auth_write` (sensitive information)
- [ ] Returns single JSON archive containing:
  - `_pg_ripple.*` table row counts
  - All non-sensitive GUC values (exclude `pg_ripple.llm_api_key` etc.)
  - 60-second Prometheus metrics snapshot
  - Extension version and HTTP companion version
- [ ] Suitable for attaching to support tickets without manual SQL

- [ ] Implement `POST /tenants/{name}/quota` (create/update quota)

### Feature 11 — Rule-Library Federation

- [ ] Implement `pg_ripple.publish_rule_library(name TEXT, endpoint_uri TEXT)` SQL function

- [ ] Fall back to primary if replica is unavailable
