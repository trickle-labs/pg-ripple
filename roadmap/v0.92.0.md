# v0.92.0 — Assessment 14 Low-Severity Polish & Hardening

> **Full technical details:** [v0.92.0-full.md](v0.92.0-full.md)

**Status: Planned** | **Scope: Medium**

> v0.92.0 closes all 39 Low-severity findings from PLAN_OVERALL_ASSESSMENT_14 across correctness documentation, security hardening, performance micro-improvements, concurrency regression tests, code quality housekeeping, standards notes, observability audits, HTTP companion polish, dependency maintenance, build system improvements, Datalog clarifications, IVM documentation, CDC edge-case handling, and documentation completeness. This is the final Assessment 14 remediation release before v1.0.0 production hardening.

---

## What is this?

After v0.89.0–v0.91.0 close all High and Medium A14 findings, this release sweeps the 39 Low findings. Individually each is a sub-hour fix; collectively they polish the codebase to the level expected before the v1.0.0 API freeze. The release also includes the worldclass aspirational notes (WC-01–WC-05) captured in the assessment's post-v1.0.0 gap analysis, filed as tracking issues for v1.1.0–v1.2.0.

## Headline outcomes

### Correctness (Low)

- [x] **CB-07** — `pagerank_lower()` / `pagerank_upper()` source comment: `-- PR-STALE-BOUNDS-01: bound = score ± (alpha^k * delta_per_iter)` added to function header in `src/pagerank/ivm.rs`.
- [x] **CB-08** — `docs/src/features/pagerank.md` "Tuning damping for your graph" subsection added: notes that the default 0.85 is calibrated for citation/web graphs; sparse social graphs typically use 0.5–0.75.
- [x] **CB-09** — `tests/pg_regress/sql/sparql_federation.sql` extended with a TLS handshake failure scenario asserting `SERVICE SILENT` swallows the error and returns an empty result set.
- [x] **CB-10** — `describe_form` GUC documentation updated: `symmetric` alias contract documented as permanent (not to be removed without deprecation notice); alias and `scbd` remain semantically identical until the SCBD spec diverges.

### Security (Low)

- [x] **SEC-06** — `audit.toml` RUSTSEC expiry review: RSA advisories expire 2026-12-01; if the RSA crate is provably unused (all TLS via `rustls-tls-native-roots`), remove the `rsa` dependency entirely. Calendar reminder added to `RELEASE.md`.
- [x] **SEC-07** — `_pg_ripple.pagerank_dirty_edges` RLS policy audited. If per-graph isolation is incomplete, add `CREATE POLICY pagerank_dirty_edges_graph_isolation ON _pg_ripple.pagerank_dirty_edges USING (graph_id = current_setting('pg_ripple.current_graph')::bigint)` mirroring `_pg_ripple.confidence`.
- [x] **SEC-08** — `pg_ripple.pagerank_find_duplicates()` volatility audited: if `VOLATILE`, change to `STABLE` so the planner can hoist it out of joins and prune by graph for multi-tenant deployments. Per-graph filter added to the function body if absent.
- [x] **SEC-09** — `.github/workflows/cargo-audit.yml` extended with `--deny unmaintained`; verify expiry enforcement is active for all four RUSTSEC ignores.

### Performance (Low)

- [x] **PERF-07** — `pg_ripple.pagerank_partition` default changed from `1` to `min(num_cpus::get(), count(named_graphs))` (computed at `_PG_init` and re-evaluated at `pagerank_run()` entry). Leaves multi-core machines idle no longer by default.
- [x] **PERF-08** — `pg:fuzzy_match()` and `pg:token_set_ratio()` annotated `IMMUTABLE` via `#[pg_extern(immutable)]` in `src/uncertain/fuzzy.rs`. Allows the planner to hoist them out of join conditions.

### Concurrency (Low)

- [x] **CON-04** — `tests/pg_regress/sql/datalog_parallel.sql` regression test: constructs a cyclic head-group dependency graph and asserts the pre-check warning `"PT3001 cyclic parallel Datalog stratification detected"` is emitted (not crashed). Verifies the A13 P13-06 fix is still in place.
- [x] **CON-05** — `tests/concurrency/confidence_subxact_rollback.sql` verifies noisy-OR aggregation rows are rolled back with their sub-transaction: `BEGIN; SAVEPOINT s1; SELECT run_inference_seminaive(); ROLLBACK TO SAVEPOINT s1; COMMIT;` — asserts confidence table is unchanged.

### Test Coverage (Low)

- [x] **TEST-06** — `benchmarks/pagerank_throughput_history.csv` created and wired to `performance_trend.yml` as a new chart. Captures per-version wall-time and convergence-iteration baselines for the 1M-edge Karate Club benchmark.

### Code Quality (Low)

- [x] **CQ-06** — `.unwrap()` / `.expect(` audit of v0.87/v0.88 additions in `src/uncertain/`, `src/pagerank/`, `pg_ripple_http/src/routing/pagerank_handlers.rs`. All 15 new occurrences (delta from A13's 35 to A14's 50) reviewed; each receives either a `// CLIPPY-OK: <reason>` annotation or conversion.
- [x] **CQ-07** — `pg_ripple_http/src/routing/admin_handlers.rs` (789 lines) and `routing/pagerank_handlers.rs` (760 lines) noted for pre-emptive split at next handler addition. Tracking issues filed. No action in v0.92.0.
- [x] **CQ-08** — `#[allow(dead_code)]` audit of v0.87/v0.88 additions: `grep -rn "#\[allow(dead_code)\]" src/uncertain/ src/pagerank/`. Each occurrence documented with a `// Q14-08: <reason>` comment following the Q13-05 convention.

### API (Low)

- [x] **API-05** — `explain_pagerank_json()` already delivered in v0.91.0 API-05. ✓
- [x] **API-06** — PT error code registry already completed in v0.91.0 API-06. ✓

### Standards (Low)

- [x] **STD-04** — `docs/src/features/pagerank.md` portability note added: `pg:pagerank()` is a pg_ripple-specific extension function; it is not portable to other SPARQL endpoints. Users of federated queries should use the full IRI form.
- [x] **STD-05** — `docs/src/features/uncertain-knowledge.md` SHACL extension note added: `sh:severityWeight` is a pg_ripple extension not in the W3C SHACL specification. Community submission to W3C SHACL Community Group considered; link to GitHub discussion.

### Observability (Low)

- [x] **OBS-04** — EXPLAIN field name standardised: `grep -r "algebra_optimi" src/ pg_ripple_http/src/`. Both `algebra_optimised` (en_GB) and `algebra_optimized` (en_US) found in source; standardise on `algebra_optimized` (en_US) to match PostgreSQL conventions.
- [x] **OBS-05** — `pg_ripple.diagnostic_report()` extended to include v0.87/v0.88 catalog: `_pg_ripple.confidence` row count, `_pg_ripple.pagerank_scores` last-computed timestamp, `_pg_ripple.pagerank_dirty_edges` queue depth, `_pg_ripple.centrality_scores` row count per metric.

### HTTP Companion (Low)

- [x] **HTTP-05** — `PG_RIPPLE_HTTP_SHUTDOWN_TIMEOUT_SECS` env var added (INT, default 30). Passed to `with_graceful_shutdown` drain timeout.
- [x] **HTTP-06** — Rate-limit response verified to include `Retry-After` header when `tower_governor` fires. If missing, add via `tower_governor`'s response transformer or a custom layer.

### Dependencies (Low)

- [x] **DEP-03** — `pgrx = "=0.18.0"` exact pin verified; pgrx 0.19 changelog checked for PG18 critical fixes. Note in `CHANGELOG.md` if any relevant fixes exist.
- [x] **DEP-04** — `spargebra` and `sparopt` versions checked against latest oxigraph release. `cargo info spargebra sparopt` output documented.
- [x] **DEP-05** — `rust-toolchain.toml` `1.95.0` pin noted as current; renovate will propose updates. No action required.

### Build System (Low)

- [x] **BUILD-03** — `justfile` recipes verified: `bump-version`, `regen-sbom`, `regen-openapi` all present (added in v0.89.0). If any are missing, add stub.
- [x] **BUILD-04** — `build.rs` audited for `SOURCE_DATE_EPOCH` support. If build timestamp is embedded without honouring `SOURCE_DATE_EPOCH`, fix for reproducible builds; document in `CONTRIBUTING.md`.
- [x] **BUILD-05** — `CONTRIBUTING.md` updated to document v0.87/v0.88 module structure (`src/uncertain/`, `src/pagerank/` submodules) and the `// Q13-05` / `// CLIPPY-OK:` / `// SAFETY:` magic-comment conventions.

### Datalog (Low)

- [x] **DL-04** — `src/datalog/magic_sets.rs` audited: magic-sets pre-condition (adornments derivable from query bindings) verified or documented. If the pre-condition is assumed but not checked, add an assertion that fires with a descriptive error message.
- [x] **DL-05** — `docs/src/features/pagerank.md` `owl:sameAs` handling note added: when `sameAs` clusters are merged, pg_ripple's canonicalization (v0.23.0) deduplicates edges before PageRank computation to avoid double-counting.
