# v0.113.0 — A16 High: bulk-load COPY path & performance tuning

**Status:** ✅ Released — 2026-05-12

## Summary

v0.113.0 addresses the remaining High-severity finding (H16-05) from Assessment 16 and delivers
five performance improvements (P4–P7): the `bulk_load_use_copy` GUC is promoted to default-on
(5–10× throughput gain for large loads), the entity-resolution embedding stage switches from O(n)
per-candidate round-trips to a single batched HNSW probe, the Bloom-filter HMAC construction
avoids redundant key expansions via `clone()`, the replication watermark constants are promoted to
GUCs, and the SSE channel capacity becomes configurable via environment variable.

## Deliverables

### H16-05 — Bulk-load COPY Path (High)

- [x] Change `BULK_LOAD_USE_COPY` default from `off` to `on` in `src/gucs/storage.rs`
- [x] Update GUC registration description in `src/gucs/registration/storage.rs`
- [x] `bulk_load_ntriples()` and `bulk_load_turtle()` use `copy_into_vp()` UNNEST-array path by
  default, sharing the helper with R2RML and CDC paths
- [x] Document 5–10× throughput gain in `docs/src/cookbook/bulk-loading.md`

### P4 — Batched Embedding HNSW Probe

- [x] Replace per-candidate embedding loop in `src/entity_resolution.rs` Stage 2
  (`run_embedding_candidates`) with a single batched `array_agg` CTE query that reduces
  SPI round-trips from O(n) to O(1) per blocking block

### P5 — HMAC Clone Optimization

- [x] Reuse a single base `HMAC` instance across all `hash_count` positions in
  `src/pprl.rs::bloom_encode` via `clone()` instead of re-keying per iteration

### P6 — Replication Batch GUCs

- [x] Add `REPLICATION_BATCH_SIZE` GUC (default 100) to `src/gucs/storage.rs`
- [x] Add `REPLICATION_BATCH_INTERVAL_MS` GUC (default 500) to `src/gucs/storage.rs`
- [x] Register both GUCs in `src/gucs/registration/storage.rs`
- [x] Update `src/replication.rs` logical apply worker to read GUCs at runtime instead of
  compile-time `const` values

### P7 — SSE Buffer Environment Variable

  `sse_channel_capacity()` function reading `PG_RIPPLE_HTTP_SSE_BUFFER` env (default 256,
