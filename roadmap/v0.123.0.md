## Notes

- The `bench_workload_result` function is a pure SQL wrapper over
  `_pg_ripple.bench_history`; no Rust code changes required
- Replica pool gauges are computed at scrape time from the live pool object —
  no background worker needed
- Rule library stream duration uses a cumulative counter (not a histogram buckets
  list) to remain consistent with the rest of the Prometheus counter pattern in this
  codebase
