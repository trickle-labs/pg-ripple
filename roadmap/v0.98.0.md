# v0.98.0 — SKOS Support, Named Bundle API & Graph Intelligence

> **Full technical details:** [v0.98.0-full.md](v0.98.0-full.md)

**Status: Planned** | **Scope: Large**

> v0.98.0 expands pg_ripple with five capabilities driven by the riverbank integration plan. It delivers: (1) a complete W3C-conformant SKOS/SKOS-XL entailment and integrity stack; (2) a formal named bundle API (`load_datalog_bundle()` / `load_shape_bundle()` / `active_datalog_bundles`) so downstream applications can reference and version rule sets by name; (3) `explain_contradiction()` — a minimal-hitting-set contradiction explainer that traces which triples and Datalog rules together produce an inconsistency; (4) a federation trust layer that tags `SERVICE`-fetched triples with `pg:sourceTrust` scores from a `federation_endpoints` registry; and (5) `coverage_map()` / `refresh_coverage_map()` — per-topic graph coverage metrics (triple count, source density, mean confidence, contradiction count) schedulable via pg_trickle.

---

## What is this?

SKOS is the W3C standard vocabulary for thesauri, taxonomies, classification schemes, and subject heading systems. pg_ripple stored SKOS data correctly but provided no automatic entailment. The riverbank planning process also identified four additional capabilities that are graph-computable entirely inside the database and belong in pg_ripple rather than an application layer.

This release delivers all five:

1. **SKOS inference** (`"skos"` rule set, 28 rules): all W3C SKOS entailments (S7–S45) — transitive hierarchies, inverse relations, symmetric `skos:related`, mapping property propagation, label and documentation sub-properties, concept-type assertions.
2. **Named bundle API**: `load_datalog_bundle(name, named_graph)` / `load_shape_bundle(name)` with an `active_datalog_bundles` catalog view including `bundle_version` — so riverbank and other consumers can assert minimum-version requirements and detect drift.
3. **SKOS-XL & SHACL integrity**: `"skosxl"` dumb-down chains (S55–S57) and a `"skos-integrity"` SHACL shape bundle (10 validators covering S9/S13/S14/S27/S37/S46 plus ISO 25964-1 structural rules). `load_shape_bundle('skos-integrity')` implicitly activates the `"skos-transitive"` bundle.
4. **`explain_contradiction()`**: traces the minimal set of triples and rules producing an inconsistency for a given subject IRI — the SHACL+Datalog analogue of `explain_pagerank()`. Greedy (fast) and exact (SAT-style) modes.
5. **Federation trust layer**: a `pg_ripple.federation_endpoints` table with per-endpoint `min_confidence`; remote `SERVICE` triples are tagged with `pg:sourceTrust` derived from the endpoint's configuration.
6. **`coverage_map()` / `refresh_coverage_map()`**: per-topic aggregate coverage — triple count, distinct source count, mean/min confidence, violation count, time range — over named graphs grouped by `skos:broader`. Results are writable as `pgc:CoverageMap` triples and schedulable via pg_trickle.
7. **Five SQL helpers**: `skos_ancestors()`, `skos_descendants()`, `skos_label()`, `skos_related()`, `skos_siblings()`.

## Headline outcomes

- [x] `pg_ripple.load_builtin_rules('skos')` installs 28 Datalog entailment rules covering all W3C SKOS entailments
- [x] `pg_ripple.load_builtin_rules('skosxl')` installs 3 SKOS-XL dumb-down property-chain rules
- [x] `pg_ripple.load_datalog_bundle(name, named_graph)` — named, versioned Datalog bundle activation
- [x] `pg_ripple.load_shape_bundle(name)` — named SHACL shape bundle activation with implicit dependency resolution
- [x] `SELECT * FROM pg_ripple.active_datalog_bundles` — catalog view with `bundle_name`, `bundle_version`, `loaded_at`
- [x] `"skos-integrity"` SHACL shape bundle with 10 validators (W3C S9/S13/S14/S27/S37/S46 + ISO 25964-1 structural rules)
- [x] `"skos-transitive"` formally named as a loadable bundle (subset of `"skos"`) for riverbank compatibility
- [x] `pg_ripple.explain_contradiction(subject_iri, named_graph, max_depth, mode)` with greedy and exact modes
- [x] `pg_ripple.explain_contradiction_json(subject_iri)` — JSONB variant
- [x] `pg_ripple.federation_endpoints` table with `min_confidence` column; remote SERVICE triples tagged with `pg:sourceTrust`
- [x] `pg_ripple.coverage_map(named_graphs, topic_predicate, top_k)` — per-topic coverage SRF
- [x] `pg_ripple.refresh_coverage_map(target_graph, named_graphs)` — writes `pgc:CoverageMap` triples
- [x] `pg_ripple.validate_skos()` — integrity report wrapper
- [x] Five new SQL helper functions: `skos_ancestors()`, `skos_descendants()`, `skos_label()`, `skos_related()`, `skos_siblings()`
- [x] pg_regress test suite: 60+ new tests
- [x] Cookbook chapter `docs/src/cookbook/skos-thesaurus.md`
- [x] Blog post `blog/skos-knowledge-organization.md`
