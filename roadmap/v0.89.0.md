# v0.89.0 — Assessment 14 Critical & High Remediation

> **Full technical details:** [v0.89.0-full.md](v0.89.0-full.md)

**Status: Released ✅** | **Scope: Large**

> v0.89.0 addresses all Critical and High findings from PLAN_OVERALL_ASSESSMENT_14, plus the highest-priority Must-fix items in the pre-v1.0.0 backlog. Key changes: delete the stale `src/gucs/registration.rs.bak` backup file and add a `.bak`-file CI lint; extend the migration-chain test with checkpoints for v0.84–v0.88 and add a structural version-sync assertion; bump `COMPATIBLE_EXTENSION_MIN` to v0.88.0 and implement `just bump-version X.Y.Z` to atomically update all nine version references on every release; add confidence noisy-OR proptest against a reference oracle; harden authentication on `/pagerank/run` to use `check_auth_write`; audit and correct v0.87/v0.88 GUC names before the v1.0.0 API freeze; enable a non-zero default rate limit in `pg_ripple_http`; and add input-length guards to the fuzzy SPARQL functions and PageRank seed array.

---

## What is this?

Assessment #14 (anchored at v0.88.0) identified 97 findings: 0 Critical, 7 High, 51 Medium, 39 Low. This release closes the 7 High findings and the 5 Must-fix items from the prioritised backlog. The most consequential finding — **DEAD-FILE-01** — is a 72 KB committed backup (`src/gucs/registration.rs.bak`) that bypasses the file-size CI lint gate, pollutes IDE search, and misleads reviewers. A second structural problem that has now recurred through three consecutive assessments (**TEST-01**) is the migration-chain test stopping at v0.83.0; v0.84–v0.88 are applied but have zero schema-level assertions. Both are single-PR fixes.

The authentication gap on `/pagerank/run` (**SEC-05**) uses `check_auth` (read-only token) for a handler that writes to `_pg_ripple.pagerank_scores`, creating an inconsistency with the rest of the write-endpoint convention. The GUC API freeze (**API-01**) ensures every name added in v0.87/v0.88 conforms to the `pg_ripple.noun_verb_unit` convention before v1.0.0 locks the API.

## Headline outcomes

- [x] **DEAD-FILE-01 (CQ-01)** — `git rm src/gucs/registration.rs.bak`; add `.bak`, `.orig`, `.swp` patterns to `.gitignore`; new `lint-no-backup-files` CI job in `ci.yml` that fails on any `*.bak` / `*.orig` / `*.swp` under `src/` or `pg_ripple_http/src/`.
- [x] **TEST-01** — `tests/test_migration_chain.sh` extended with checkpoint assertions for v0.84 (no DDL), v0.85 (no DDL), v0.86 (no DDL), v0.87 (`_pg_ripple.confidence`, `shacl_score_log`, `confidence_stmt_idx`, GIN index), v0.88 (`pagerank_scores`, `pagerank_dirty_edges`, `centrality_scores`, BRIN index). Structural assertion added at the end of the script: `assert_true "highest checkpoint matches highest migration" "..."` — fails CI automatically when a new migration ships without a corresponding checkpoint, eliminating the recurring partial-fix pattern.
- [x] **HTTP-COMPAT-01** — `COMPATIBLE_EXTENSION_MIN` bumped from `"0.87.0"` to `"0.88.0"` in `pg_ripple_http/src/main.rs:38`. Update comment to document the new minimum.
- [x] **ROAD-02 (just bump-version)** — `just bump-version X.Y.Z` recipe implemented in `justfile`. Atomically updates: `Cargo.toml` version, `pg_ripple_http/Cargo.toml` version, `pg_ripple.control` `default_version`, `pg_ripple_http/src/main.rs` `COMPATIBLE_EXTENSION_MIN`, `docker-compose.yml` image tags, creates `sql/pg_ripple--<prev>--<new>.sql` migration stub, creates `CHANGELOG.md` stub section. Includes a dry-run mode (`just bump-version-dry X.Y.Z`) that prints the proposed changes without writing them.
- [x] **CB-01** — `tests/proptest/confidence_algebra.rs` proptest suite verifying noisy-OR algebraic identities: commutativity, associativity, monotonicity (`c1 ≤ c2 ⟹ noisy_or(c1, c3) ≤ noisy_or(c2, c3)`), idempotence (`noisy_or(c, c) = 1 - (1-c)^2`), identity (`noisy_or(0.0, c) = c`), absorbing element (`noisy_or(1.0, c) = 1.0`). Tests compare pg_ripple's in-database computation via `pg_ripple.compute_confidence(...)` against an in-process Rust reference. Uses the `tests/proptest/ntriples_oxigraph.rs` harness shape.
- [x] **SEC-05** — `pg_ripple_http/src/routing/pagerank_handlers.rs` `/pagerank/run`, `/pagerank/vacuum-dirty`, `/centrality/run`, and `/pagerank/find-duplicates` handlers updated to call `check_auth_write` instead of `check_auth`. Read-only endpoints (`/pagerank/results`, `/pagerank/status`, `/pagerank/export`, `/pagerank/explain/*`, `/pagerank/queue-stats`, `/centrality/results`) continue to use `check_auth`. Policy documented in `docs/src/operations/security.md`.
- [x] **API-01** — GUC name lint pass over all v0.87/v0.88 GUCs against the `pg_ripple.noun_verb_unit` convention: `pagerank_`, `pagerank_confidence_`, `prob_datalog_`, `fuzzy_`, `conf_` prefixes audited. Any violating GUC name gets a deprecated alias (backward-compatible) and the canonical name; deprecation scheduled for v1.0.0 removal. `src/gucs/pagerank.rs` and `src/gucs/observability.rs` updated. Regression tests added for each deprecated alias.
- [x] **SEC-01** — `PG_RIPPLE_HTTP_RATE_LIMIT` default changed from `0` (disabled) to `100` (100 req/s). Documentation updated: "set to `0` to disable rate limiting when behind a reverse proxy that handles it". Operators with existing explicit `PG_RIPPLE_HTTP_RATE_LIMIT=0` are unaffected.
- [x] **SEC-02** — `pg_ripple.fuzzy_max_input_length` GUC (INT, default 4096, range 1–65536). `pg:fuzzy_match()` and `pg:token_set_ratio()` raise `PT0303` with message `"fuzzy SPARQL input exceeds pg_ripple.fuzzy_max_input_length characters — truncate input or raise the GUC"` when either argument exceeds the limit. Registered in `src/gucs/registration/security.rs`.
- [x] **SEC-03** — `pg_ripple.pagerank_max_seeds` GUC (INT, default 1024, range 1–1048576). `pagerank_run(..., seed_iris TEXT[])` raises `PT0411` when `array_length(seed_iris, 1) > pagerank_max_seeds`. Registered in `src/gucs/registration/pagerank.rs`.
- [x] **SEC-04** — `pg_ripple.export_pagerank()` routes all output through the existing IRI serializer in `src/export/turtle.rs` (shared with `export_turtle()`). Removes the ad-hoc string interpolation of the `topic` parameter; malicious IRIs containing `>`, `<`, `"`, `\`, or ASCII control characters are now percent-encoded per RFC 3987. Regression test with adversarial `topic` IRI added to `tests/pg_regress/sql/pagerank.sql`.
- [x] **CB-03 (fuzzy_match pg_trgm diagnostic)** — `pg:fuzzy_match()` and `pg:token_set_ratio()` now raise `PT0301` with message `"pg_ripple fuzzy SPARQL requires pg_trgm — install it with: CREATE EXTENSION IF NOT EXISTS pg_trgm;"` when the extension is not installed. Previously returned a generic error.
- [x] **Migration script** — `sql/pg_ripple--0.88.0--0.89.0.sql`: no schema changes; comment header documents security hardening (SEC-02, SEC-03, SEC-04) and GUC additions (fuzzy_max_input_length, pagerank_max_seeds). GUC defaults applied via `ALTER SYSTEM SET` are documented.
