## 1. DEAD-FILE-01 / CQ-01 — Stale Backup File

**Finding**: `src/gucs/registration.rs.bak` (72,962 bytes) is committed to the source tree. The `.bak` extension bypasses the file-size CI lint gate (which checks `*.rs` files). It pollutes IDE symbol search, grep results, and code review. Cargo ignores it, so it has no runtime impact, but it is misleading.

**Implementation**:
```bash
git rm src/gucs/registration.rs.bak
```

Add to `.gitignore`:
```
*.bak
*.orig
*.swp
*.swo
```

Add to `.github/workflows/ci.yml`:
```yaml
- name: Lint — no backup files in src/
  run: |
    count=$(find src pg_ripple_http/src -name '*.bak' -o -name '*.orig' -o -name '*.swp' | wc -l)
    if [ "$count" -gt 0 ]; then
      echo "ERROR: backup files found in source tree:"
      find src pg_ripple_http/src -name '*.bak' -o -name '*.orig' -o -name '*.swp'
      exit 1
    fi
```

**Effort**: S (< 30 min). No migration script changes required.

## 4. ROAD-02 — `just bump-version X.Y.Z`

**Finding**: The HTTP companion `COMPATIBLE_EXTENSION_MIN` lag is recurring (A13 6-release gap, A14 1-release gap). The root cause is a manual multi-file update process. A `just bump-version` recipe eliminates the class.

**Implementation** (add to `justfile`):
```makefile
# Bump all version references atomically.
# Usage: just bump-version 0.89.0
# Dry-run: just bump-version-dry 0.89.0
bump-version NEW_VERSION:
    #!/usr/bin/env bash
    set -euo pipefail
    OLD=$(grep '^version' Cargo.toml | head -1 | grep -oE '[0-9]+\.[0-9]+\.[0-9]+')
    echo "Bumping $OLD → {{NEW_VERSION}}"
    # 1. Cargo.toml
    sed -i '' "s/^version = \"$OLD\"/version = \"{{NEW_VERSION}}\"/" Cargo.toml
    # 2. pg_ripple_http/Cargo.toml
    sed -i '' "s/^version = \"$OLD\"/version = \"{{NEW_VERSION}}\"/" pg_ripple_http/Cargo.toml
    # 3. pg_ripple.control
    sed -i '' "s/default_version = '$OLD'/default_version = '{{NEW_VERSION}}'/" pg_ripple.control
    # 4. pg_ripple_http COMPATIBLE_EXTENSION_MIN
    sed -i '' "s/COMPATIBLE_EXTENSION_MIN: \&str = \"$OLD\"/COMPATIBLE_EXTENSION_MIN: \&str = \"{{NEW_VERSION}}\"/" pg_ripple_http/src/main.rs
    # 5. docker-compose.yml image tags
    sed -i '' "s/:$OLD/:{{NEW_VERSION}}/g" docker-compose.yml
    # 6. Create migration stub
    touch "sql/pg_ripple--$OLD--{{NEW_VERSION}}.sql"
    echo "-- Migration $OLD → {{NEW_VERSION}}: add SQL changes here" >> "sql/pg_ripple--$OLD--{{NEW_VERSION}}.sql"
    # 7. CHANGELOG stub
    echo "\n## [{{NEW_VERSION}}] — $(date +%Y-%m-%d)\n\n### Added\n\n### Fixed\n\n### Changed\n" | \
      sed -i '' "/^## \[Unreleased\]/r /dev/stdin" CHANGELOG.md || true
    echo "Done. Review changes with: git diff"

bump-version-dry NEW_VERSION:
    #!/usr/bin/env bash
    OLD=$(grep '^version' Cargo.toml | head -1 | grep -oE '[0-9]+\.[0-9]+\.[0-9]+')
    echo "[dry-run] Would bump $OLD → {{NEW_VERSION}} in:"
    echo "  Cargo.toml, pg_ripple_http/Cargo.toml, pg_ripple.control,"
    echo "  pg_ripple_http/src/main.rs (COMPATIBLE_EXTENSION_MIN),"
    echo "  docker-compose.yml (image tags)"
    echo "  Create: sql/pg_ripple--$OLD--{{NEW_VERSION}}.sql"
    echo "  Update: CHANGELOG.md [Unreleased] section"
```

Add a CI job to validate that both `Cargo.toml` versions match and match `pg_ripple.control`:
```yaml
# .github/workflows/ci.yml — added step:
- name: Lint — version sync
  run: |
    ext_ver=$(grep '^version' Cargo.toml | head -1 | grep -oE '[0-9]+\.[0-9]+\.[0-9]+')
    http_ver=$(grep '^version' pg_ripple_http/Cargo.toml | head -1 | grep -oE '[0-9]+\.[0-9]+\.[0-9]+')
    ctrl_ver=$(grep "^default_version" pg_ripple.control | grep -oE '[0-9]+\.[0-9]+\.[0-9]+')

      exit 1
    fi
    echo "Version sync OK: $ext_ver"
```

## 5. CB-01 — Confidence Noisy-OR Proptest

**Finding**: v0.87.0 noisy-OR composition (`noisy_or(c1, c2) = 1 - (1-c1)*(1-c2)`) has no property-based test against an oracle. Without this, refactors of the confidence join order or the SQL aggregation can drift silently.

**Implementation**: `tests/proptest/confidence_algebra.rs`

```rust
use proptest::prelude::*;

proptest! {
    /// Commutativity: noisy_or(a, b) == noisy_or(b, a)
    #[test]
    fn noisy_or_commutative(a in 0.0f64..=1.0, b in 0.0f64..=1.0) {
        let ab = reference_noisy_or(a, b);
        let ba = reference_noisy_or(b, a);
        prop_assert!((ab - ba).abs() < 1e-12);
    }

    /// Identity: noisy_or(0.0, c) == c
    #[test]
    fn noisy_or_identity(c in 0.0f64..=1.0) {
        let result = reference_noisy_or(0.0, c);
        prop_assert!((result - c).abs() < 1e-12);
    }

    /// Absorbing element: noisy_or(1.0, c) == 1.0
    #[test]
    fn noisy_or_absorbing(c in 0.0f64..=1.0) {
        let result = reference_noisy_or(1.0, c);
        prop_assert!((result - 1.0).abs() < 1e-12);
    }

    /// Monotonicity: if a1 <= a2 then noisy_or(a1, b) <= noisy_or(a2, b)
    #[test]
    fn noisy_or_monotone(a1 in 0.0f64..=1.0, a2 in 0.0f64..=1.0, b in 0.0f64..=1.0) {
        let (lo, hi) = if a1 <= a2 { (a1, a2) } else { (a2, a1) };
        let r_lo = reference_noisy_or(lo, b);
        let r_hi = reference_noisy_or(hi, b);
        prop_assert!(r_lo <= r_hi + 1e-12);
    }
}

fn reference_noisy_or(a: f64, b: f64) -> f64 {
    1.0 - (1.0 - a) * (1.0 - b)
}
```

The in-database comparison uses an integration test in `tests/pg_regress/sql/confidence_proptest.sql` that runs 1000 random (a, b) pairs through `pg_ripple.compute_confidence()` and asserts each matches the reference formula within 1e-10.

**Effort**: M (2–4 hours).

## 7. API-01 — GUC Name Audit (v0.87/v0.88)

**Finding**: 22 new GUCs were added in v0.88.0, 3 in v0.87.0. Any violating names must be corrected with backward-compatible aliases before v1.0.0 locks the API.

**Convention** (`GUC-NAME-01`): `pg_ripple.<noun>_<verb_or_qualifier>_<unit>`, all lowercase, underscores, no abbreviations except established ones (`iri`, `ivm`, `rls`).

**Audit results** (non-conforming names found):
- `pg_ripple.katz_alpha` → rename to `pg_ripple.katz_attenuation_factor` (with alias `katz_alpha`)
- `pg_ripple.prob_datalog_max_iterations` — acceptable abbreviation `prob_datalog` for "probabilistic Datalog"
- `pg_ripple.pagerank_khop_limit` → rename to `pg_ripple.pagerank_incremental_khop_limit` for clarity

**Migration**: deprecated aliases registered via `DefineCustomRealVariable` / `DefineCustomIntVariable` with `GUC_DEPRECATED` flag (PostgreSQL 17+ feature) pointing to canonical names; `WARNING` emitted on use. Aliases removed in v1.0.0.

**Effort**: S (audit) + M (renames with aliases) = M total.
