# v0.95.0 — A15 Medium: Correctness, Security, Storage

**Status**: Planned → In Progress

## Summary

This release delivers the A15 medium-priority items covering correctness, security, and storage: replace `unreachable!()` panics with user-visible errors, add DNS rebinding protection for federation, add an SQL-drop event trigger for replication-slot cleanup on extension uninstall, improve SSE error redaction, add scheduled dictionary VACUUM, add property-path+OPTIONAL+vp_rare regression tests, add NaN/Inf rejection for confidence values, fold the schema-generation counter into the plan cache key, and route SPARQL ADD/COPY/MOVE through the full update pipeline.

## Deliverables

- [x] M15-01: Replace `unreachable!()` in `pagerank/export.rs` and `pagerank/centrality.rs` with `pgrx::error!` + CI zero-unreachable-in-production lint
- [x] M15-02: Resolve-once DNS rebinding fix in `federation/policy.rs` — validate every resolved IP against the SSRF blocklist, connect to resolved IP with pinned `Host:` header
- [x] M15-03: `sql_drop` event trigger for `DROP EXTENSION` replication-slot cleanup
- [x] M15-04: `redacted_error()` for SSE initialisation error paths in `stream.rs`
- [x] M15-07: Scheduled `VACUUM ANALYZE _pg_ripple.dictionary` after bulk encode above threshold GUC, plus `autovacuum_scale_factor` `reloptions` on the dictionary table
- [x] M15-08: Explicit regression tests for `OPTIONAL` + property paths inside `GRAPH {}` with `vp_rare` predicates
- [x] M15-09: NaN/Inf/out-of-range rejection in `load_triples_with_confidence()` and confidence paths — raise PT0301
- [x] M15-10: Fold `_pg_ripple.schema_generation` counter into plan cache key, bump on every VP promotion and `ensure_vp_table()` call
- [x] M15-12: Integrate `ADD`/`COPY`/`MOVE` SPARQL Update operations through the main UPDATE pipeline with CDC and SHACL queue integration tests
