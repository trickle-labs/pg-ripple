# v0.100.0 — Proof Trees & Justification Infrastructure

**Theme**: Proof trees & justification infrastructure.

> **In plain language:** When pg_ripple derives a new fact using Datalog rules, it can now record
> *why* it believed that fact — which rules fired, which base triples satisfied the rule bodies —
> and present that chain of reasoning as a structured JSON proof tree. This is the single most
> important missing piece for using pg_ripple as a true expert system: the ability to answer
> "why did you conclude this?" in a machine-readable, auditable form.
>
> **Effort estimate: 1–2 person-weeks**

## Prerequisites

- v0.99.2 released and all tests passing.

## Deliverables

- [x] `_pg_ripple.derivations` table recording `(derived_sid, rule_name, rule_set, antecedent_sids BIGINT[], created_at TIMESTAMPTZ)` for every Datalog-derived fact
- [x] `pg_ripple.record_derivations` GUC (default `off`) gates the overhead — recording only happens when this is `on`
- [x] Recording wired into the semi-naive inference engine: after each rule fires, antecedent SIDs are captured and stored
- [x] `pg_ripple.justify(subject TEXT, predicate TEXT, object TEXT) → JSONB` SQL function returning the full backward-chaining proof tree
- [x] Recursive derivation-graph walker with cycle protection (visited-SID set prevents infinite loops)
- [x] Batch dictionary decode for human-readable IRI labels in proof output (subject, predicate, object as decoded strings)
- [x] `pg_ripple.vacuum_derivations() → BIGINT` function to remove orphan derivation rows (where derived_sid no longer exists in any VP table)
- [x] Derivations survive DRed retraction cleanly: `run_dred_on_delete` removes derivation rows for deleted SIDs; subsequent `vacuum_derivations()` cleans residual orphans
- [x] Migration script `sql/pg_ripple--0.99.2--0.100.0.sql`
- [x] pg_regress tests: `tests/pg_regress/sql/v0100_features.sql` covering derivation recording, proof-tree structure, cycle protection, and `vacuum_derivations()`

## Exit Criteria

- `cargo pgrx regress pg18` passes all 253+ tests including `v0100_features`
- `justify()` returns a valid JSONB proof tree for a simple inferred fact
- `justify()` returns `null` for a base (non-inferred) fact
- `justify()` handles cycles safely without infinite recursion
- `vacuum_derivations()` removes orphan rows and returns the count removed
- `record_derivations = off` (default) has zero overhead: derivations table remains empty
