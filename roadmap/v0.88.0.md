# v0.88.0 — Datalog-Native PageRank

> **Full technical details:** [v0.88.0-full.md](v0.88.0-full.md)

**Status: Planned** | **Scope: Very Large**

> v0.88.0 implements iterative PageRank entirely inside pg_ripple's Datalog engine, leveraging aggregation in rule bodies (Datalog^agg), subsumptive tabling for convergence-aware early termination, and magic-sets transformation for goal-directed partial-graph evaluation. A `pg:pagerank()` SPARQL function surfaces ranked nodes inline; `pg_ripple.pagerank_run()` provides a SQL entry point with full parameter control; a materialized `pagerank_scores` VP-aligned view enables fast top-N lookups; `pg:topN()` and sketch-based `pg:topN_approx()` serve exact and approximate ranked windows; and a **pg-trickle incremental refresh path** propagates rank deltas using bounded K-hop local push so that edge insertions and deletions update scores in milliseconds without a full recompute. A comprehensive enrichment layer adds: confidence-weighted edges (v0.87.0 integration), topic-sensitive multi-run scoring, edge-weight predicates, reverse/in-degree direction, temporal decay, SHACL constraint-aware ranking, predicate-scoped personalization, score-explanation trees, staleness confidence bounds, graph-partitioned parallel computation, selective recomputation, standard-format export (Turtle/JSON-LD/CSV/N-Triples), federation-aware planner classification, and four alternative centrality measures (betweenness, closeness, eigenvector, Katz).

---

## What is this?

PageRank assigns a numerical importance score to every node in a directed graph based on the structure of incoming links. In an RDF triplestore the graph is implicit: any predicate whose values are IRIs defines edges. v0.88.0 makes PageRank a first-class feature by expressing the iterative computation as Datalog rules and routing it through pg_ripple's existing optimizer — magic sets, tabling, parallel stratum evaluation, and semi-naive evaluation all apply automatically.

Full recomputes are practical for batch workloads, but a single new edge should not force re-ranking the entire graph. v0.88.0 integrates with pg-trickle's Z-set differential dataflow to propagate rank deltas locally: when a VP delta table fires, a pg-trickle stream trigger enqueues the changed edge, and a background IVM worker applies bounded K-hop rank propagation (decaying at rate $d^k$, terminating when the delta falls below the convergence threshold). The `pagerank_scores` table gains `stale` and `stale_since` columns so callers can distinguish exact from approximated scores.

Beyond the core algorithm, v0.88.0 adds a full suite of enrichments: confidence-weighted edges (integrating with v0.87.0's uncertain knowledge engine), topic-sensitive multi-run scoring, arbitrary edge-weight predicates, reverse/in-degree ranking for hub-and-authority decomposition, temporal decay for time-series graphs, SHACL constraint-aware exclusion and importance hints, score-explanation trees (`pg_ripple.explain_pagerank()`), approximate sketch-based top-K, graph partitioning for parallel computation, incremental update metrics, standard-format export (Turtle/JSON-LD/CSV/N-Triples), federation-aware planner classification, four alternative centrality measures (betweenness, closeness, eigenvector, Katz), and six cross-version synergies that deepen the v0.87.0 integration: confidence-attenuated K-hop propagation (PR-TRICKLE-CONF-01), probabilistic PageRank via `@weight` Datalog rules (PR-PROB-DATALOG-01), centrality-guided entity deduplication (PR-ENTITY-RESOLUTION-01), source-trust-weighted eigenvector centrality (PR-TRUST-EIGEN-01), confidence-gated federation edges (PR-FED-CONF-01), and temporal authority via Katz centrality (PR-KATZ-TEMPORAL-01). All enrichments are optional and GUC-controlled.

## Headline outcomes

- [x] **PR-DATALOG-01** — Built-in Datalog rule set for PageRank: `pg_ripple.pagerank_rules` GUC selects the active edge predicate(s) (comma-separated IRI list, default `''` meaning all object-valued predicates). Rules emit `pg:hasPageRank` triples on convergence. Controlled by `pg_ripple.pagerank_enabled` GUC (bool, default off).
- [x] **PR-ITER-01** — Convergence-aware early termination: `pg_ripple.pagerank_max_iterations` GUC (INT, default 100), `pg_ripple.pagerank_convergence_delta` GUC (FLOAT8, default 0.0001). Tabling detects a fixed point and halts without exhausting the iteration cap. `explain_inference()` reports `iterations`, `converged`, and `final_delta` for PageRank strata.
- [x] **PR-DAMPING-01** — Configurable damping factor via `pg_ripple.pagerank_damping` GUC (FLOAT8, default 0.85). Dangling-node redistribution redistributes rank mass from nodes with zero out-degree uniformly across all nodes in the same component (controlled by `pg_ripple.pagerank_dangling_policy` GUC: `'redistribute'` | `'ignore'`, default `'redistribute'`).
- [x] **PR-BLANK-01** — Blank-node exclusion policy: `pg_ripple.pagerank_include_blank_nodes` GUC (bool, default false). When off, blank nodes are removed from the edge set before computation and excluded from result triples.
- [x] **PR-PERSONAL-01** — Personalized PageRank: `pg_ripple.pagerank_run(seed_iris TEXT[], bias FLOAT8 DEFAULT 0.15, predicate_filter TEXT[] DEFAULT NULL, ...)` overload accepts a seed-node array and replaces the uniform teleportation vector with a bias concentrated on the seed set. The optional `predicate_filter` restricts which edge predicates carry the personalization bias, enabling per-relationship personalization (e.g., bias flows only through `foaf:knows` edges).
- [x] **PR-SPARQL-FN-01** — `pg:pagerank(?node)` SPARQL function compiles to a lookup against the materialized `pagerank_scores` view (or triggers an on-demand run if the view is stale and `pg_ripple.pagerank_on_demand = on`). Usable in `FILTER`, `BIND`, `ORDER BY`, and `HAVING`.
- [x] **PR-TOPN-01** — `pg:topN(?score, N)` aggregate function returns the top N nodes by descending score as a JSON array. Compiles to `ORDER BY … LIMIT` with BRIN-index range scan on `pagerank_scores`.
- [x] **PR-SQL-FN-01** — `pg_ripple.pagerank_run(edge_predicates TEXT[] DEFAULT NULL, damping FLOAT8 DEFAULT 0.85, max_iterations INT DEFAULT 100, convergence_delta FLOAT8 DEFAULT 0.0001, graph_uri TEXT DEFAULT NULL, direction TEXT DEFAULT 'forward', edge_weight_predicate TEXT DEFAULT NULL, topic TEXT DEFAULT NULL, decay_rate FLOAT8 DEFAULT 0.0, temporal_predicate TEXT DEFAULT NULL, seed_iris TEXT[] DEFAULT NULL, bias FLOAT8 DEFAULT 0.15, predicate_filter TEXT[] DEFAULT NULL) RETURNS TABLE (node_iri TEXT, score FLOAT8, score_lower FLOAT8, score_upper FLOAT8, iterations INT, converged BOOL, stale BOOL, topic TEXT)` SQL function. Writes results to the `pagerank_scores` VP-aligned view transactionally. Runs in parallel strata when `pg_ripple.pagerank_partition = on`.
- [x] **PR-VIEW-01** — `pagerank_scores (node BIGINT, topic TEXT NOT NULL DEFAULT '', score FLOAT8, score_lower FLOAT8, score_upper FLOAT8, computed_at TIMESTAMPTZ, iterations INT, converged BOOL, stale BOOL NOT NULL DEFAULT false, stale_since TIMESTAMPTZ, PRIMARY KEY (node, topic))` table in `_pg_ripple` schema, populated by `pagerank_run()` (exact) or the pg-trickle IVM worker (incremental). BRIN index on `(topic, score DESC)` for top-N range scans. `stale = true` marks nodes whose scores were updated by K-hop local push rather than a full recompute. `score_lower`/`score_upper` carry theoretical error bounds from K-hop incremental updates; both equal `score` after a full recompute. Exposed as a SPARQL-queryable named graph `pg:pagerank` via a synthetic VP entry. `pg:pagerank(?node)` returns the score for the default topic; `pg:pagerank(?node, ?topic)` selects a specific topic run. Callers may add `FILTER(!pg:isStale(?node))` to restrict to exact scores.
- [x] **PR-MAGIC-01** — Magic-sets integration: when `pg:pagerank(?node)` appears in a query with a bound `?node`, the Datalog optimizer applies demand transformation to restrict the PageRank computation to the subgraph reachable from or to the seed node. `explain_sparql()` reports `pagerank_scope: partial` vs `pagerank_scope: full`.
- [x] **PR-TRICKLE-01** — pg-trickle incremental refresh: when `pg_ripple.pagerank_incremental = on` (bool, default off; requires pg-trickle ≥ 0.5) and `pg_ripple.trickle_integration = on`, each VP delta-table insert/delete fires a pg-trickle Z-set stream trigger that enqueues `(source, target, +1/-1)` into `_pg_ripple.pagerank_dirty_edges`. A background IVM worker drains the queue, applies bounded K-hop rank propagation (up to `pg_ripple.pagerank_khop_limit` hops, default 30; terminates early when the delta magnitude falls below `pagerank_convergence_delta`), writes updated scores to `pagerank_scores`, and sets `stale = true` / `stale_since = now()` for affected rows. A scheduled full `pagerank_run()` (configurable via `pg_ripple.pagerank_refresh_schedule` cron expression, default `'0 3 * * *'`) resets `stale = false`. `pg_ripple.vacuum_pagerank_dirty()` helper purges processed entries. Graceful degradation: if pg-trickle is absent, `PR-TRICKLE-01` functions return `PT0408` and the rest of PageRank is unaffected.
- [x] **PR-TRICKLE-CONF-01** — Confidence-attenuated K-hop propagation: when both `pagerank_incremental = on` and `pagerank_confidence_weighted = on` are active, each rank delta propagated by the PR-TRICKLE-01 IVM worker is additionally multiplied by the edge confidence from `_pg_ripple.confidence` at each hop, so that a 0.8-confidence edge carries 80% of its normal delta weight per hop. Low-confidence subgraphs reach the convergence threshold in fewer hops, reducing unnecessary re-ranking of distant nodes triggered by uncertain edges. Controlled by `pg_ripple.pagerank_trickle_confidence_attenuation` GUC (bool, default on when both parent GUCs are on). PT0421 fires if the attenuated delta drops below `pagerank_convergence_delta` in fewer than 2 hops, indicating a probable data quality issue rather than genuine convergence. Requires v0.87.0 uncertain knowledge engine.
- [x] **PR-CONF-01** — Confidence-weighted PageRank: when `pg_ripple.pagerank_confidence_weighted = on` (bool, default off) and the v0.87.0 uncertain knowledge engine is active, each edge weight is multiplied by the confidence value from `_pg_ripple.confidence` for that statement. Edges with no confidence row are treated as weight 1.0. `pg_ripple.pagerank_confidence_default` GUC (FLOAT8, default 1.0) controls the fallback. Verified to survive the HTAP merge cycle.
- [x] **PR-PROB-DATALOG-01** — Probabilistic PageRank rules: when `pg_ripple.pagerank_probabilistic = on` (bool, default off; requires v0.87.0 `probabilistic_datalog = on`), the core Datalog rule set is annotated with `@weight(pg:confidence(?s, ?p, ?o))` so that each rule firing carries the confidence of the triggering edge and scores are combined per-node using noisy-OR (consistent with v0.87.0 semantics). Results are stored in `pagerank_scores` under `topic = '__probabilistic'` with uncertainty intervals in `score_lower`/`score_upper` reflecting the combined confidence-weighted uncertainty. `pg:pagerank(?node, '__probabilistic')` retrieves probabilistic scores; `pg:pagerank_lower(?node)` and `pg:pagerank_upper(?node)` expose the confidence interval. Enables uncertainty quantification in entity importance: callers can ask not just "which node ranks highest" but "how certain are we of that ranking".
- [x] **PR-TOPIC-01** — Topic-sensitive PageRank: `pg_ripple.pagerank_run(..., topic TEXT DEFAULT NULL)` stores results with a `topic` key in `pagerank_scores`. Multiple topic runs coexist; `pg:pagerank(?node, ?topic)` two-argument SPARQL form selects the topic-specific score. `pg_ripple.pagerank_run_topics(topics TEXT[][])` runs all listed topics in parallel worker strata.
- [x] **PR-WEIGHT-01** — Edge-weight parameterization: `pg_ripple.pagerank_run(..., edge_weight_predicate TEXT DEFAULT NULL)`. When set, the Datalog rules look up `(s <edge_weight_predicate> ?w)` per edge and use `?w` (FLOAT8) as the edge weight. Missing or non-numeric `?w` falls back to 1.0. PT0411 fires for negative weights.
- [x] **PR-REVERSE-01** — Reverse / in-degree PageRank: `pg_ripple.pagerank_run(..., direction TEXT DEFAULT 'forward')`. When `direction = 'reverse'`, Datalog rules operate on the transpose graph, ranking nodes pointed to by high-scoring nodes (authority ranking). Enables HITS-style hub/authority decomposition via two separate runs. PT0412 fires for unrecognised direction values.
- [x] **PR-EXPLAIN-SCORE-01** — Score explanation: `pg_ripple.explain_pagerank(node_iri TEXT, top_k INT DEFAULT 5) RETURNS TABLE (depth INT, contributor_iri TEXT, contribution FLOAT8, path TEXT)` returns the top-K incoming contributor chain. `pg:explainPagerank(?node)` SPARQL function returns the explanation as a JSON literal. Output is deterministic for a given `pagerank_scores` snapshot. PT0413 fires for unknown nodes.
- [x] **PR-STALE-BOUNDS-01** — Confidence bounds on approximate scores: after each K-hop incremental update the IVM worker writes the theoretical error band to `score_lower` / `score_upper` in `pagerank_scores`. Full `pagerank_run()` sets lower = upper = score. `pg:pagerank_lower(?node)` and `pg:pagerank_upper(?node)` SPARQL functions expose the bounds.
- [x] **PR-IVM-METRICS-01** — Incremental update metrics: `pg_ripple.pagerank_queue_stats() RETURNS TABLE (queued_edges BIGINT, max_delta FLOAT8, oldest_enqueue TIMESTAMPTZ, estimated_drain_seconds FLOAT8)` SQL function. Prometheus counter `pg_ripple_pagerank_dirty_edges_total` and gauge `pg_ripple_pagerank_queue_depth` exported via `/metrics`. `pg_ripple.pagerank_queue_warn_threshold` GUC (INT, default 100000) logs a WARNING when exceeded.
- [x] **PR-SKETCH-01** — Sketch-based top-K: `pg:topN_approx(?node, K, error_threshold FLOAT8 DEFAULT 0.01)` SPARQL aggregate uses a space-saving sketch to return approximate top-K nodes without a full sort. `pg_ripple.topn_approx_run(K INT, error_threshold FLOAT8 DEFAULT 0.01) RETURNS TABLE (rank INT, node_iri TEXT, score_lower FLOAT8, score_upper FLOAT8)` SQL function. PT0414 fires if `error_threshold` ≤ 0 or > 1.
- [x] **PR-PARTITION-01** — Graph partitioning for parallel computation: when `pg_ripple.pagerank_partition = on` (bool, default off), a pre-pass Datalog rule detects weakly connected components; each component is ranked independently with a damped cross-component teleportation term. Components are evaluated in parallel worker strata (v0.35.0 mechanism). `explain_inference()` reports component count and largest component size.
- [x] **PR-SELECTIVE-01** — Selective edge recomputation: `pg_ripple.pagerank_selective_threshold` GUC (FLOAT8, default 0.0) sets a minimum score below which dirty nodes are queued but not immediately re-propagated; they are flushed during the scheduled full recompute. Reduces IVM worker CPU on scale-free graphs. Affected nodes are marked `stale = true` immediately.
- [x] **PR-TEMPORAL-01** — Temporal decay for time-series graphs: `pg_ripple.pagerank_run(..., decay_rate FLOAT8 DEFAULT 0.0, temporal_predicate TEXT DEFAULT NULL)`. When set, edge weight is multiplied by `exp(-decay_rate * age_in_days(t))` where `t` is the object of `(s <temporal_predicate> ?t)`. Edges with no timestamp receive weight 1.0. PT0415 fires if `temporal_predicate` has non-temporal objects.
- [x] **PR-SHACL-01** — Constraint-aware ranking: SHACL property shapes may declare `sh:importance xsd:decimal` (multiplier on incoming rank mass from that predicate, default 1.0) and `sh:excludeFromRanking xsd:boolean` (removes the node from computation). Additionally, nodes whose source graph scores below `pg_ripple.pagerank_shacl_threshold` (FLOAT8 GUC, default 0.5, range [0.0, 1.0]) on v0.87.0's `shacl_score()` function are automatically excluded from the PageRank computation before the first iteration; this prevents low-quality subgraphs from distorting importance scores. `pg_ripple.pagerank_run()` includes `shacl_excluded_nodes INT` in its return set. `explain_pagerank()` reports `shacl_excluded: true` and `shacl_score: <value>` for excluded nodes. PT0422 fires if `pagerank_shacl_threshold` is outside [0.0, 1.0]. Requires both the v0.87.0 SHACL engine and `pagerank_shacl_threshold > 0.0`; when `pagerank_shacl_threshold = 0.0` (disabled), the `sh:importance`/`sh:excludeFromRanking` shape hints still apply via the Datalog rule set alone.
- [x] **PR-EXPORT-01** — Export to standard formats: `pg_ripple.export_pagerank(format TEXT DEFAULT 'turtle', top_k INT DEFAULT NULL, topic TEXT DEFAULT NULL) RETURNS TEXT` SQL function. Supported formats: `'turtle'` (`<node> pg:hasPageRank ?score`), `'jsonld'` (JSON-LD 1.1 with `@annotation` confidence blocks), `'csv'` (node_iri, score, stale), `'ntriples'`. REST endpoint `GET /pagerank/export?format=csv&top_k=1000` streams results. PT0417 fires for unsupported format strings.
- [x] **PR-FED-01** — Federation-aware PageRank: `pg:pagerank()`, `pg:pagerank_lower()`, `pg:pagerank_upper()`, `pg:topN()`, `pg:topN_approx()`, `pg:explainPagerank()`, and `pg:centrality()` classified as local-only in the federation planner. Optional blend mode: `pg_ripple.pagerank_federation_blend = on` fetches edge triples from remote SERVICE endpoints into a local temporary graph before `pagerank_run()`, enabling global-graph ranking across federated sources. PT0418 fires if a blend-mode endpoint is unreachable.
- [x] **PR-FED-CONF-01** — Confidence-gated federation edges: when federation blend mode is active (`pagerank_federation_blend = on`), `pg_ripple.federation_minimum_confidence` GUC (FLOAT8, default 0.5, range [0.0, 1.0]) filters remote SERVICE endpoint edges before they are loaded into the local temporary graph for `pagerank_run()`. Remote edges with no confidence annotation are treated as weight equal to `pagerank_confidence_default`. This prevents low-confidence triples from external sources from distorting global-graph importance scores. `explain_sparql()` reports `federation_confidence_filter: <threshold>` when active. PT0423 fires if `federation_minimum_confidence` is outside [0.0, 1.0]. Requires v0.87.0 uncertain knowledge engine; ignored (with a DEBUG-level log) when `pagerank_confidence_weighted = off`.
- [x] **PR-CENTRALITY-01** — Alternative centrality measures: `pg_ripple.centrality_run(metric TEXT, edge_predicates TEXT[] DEFAULT NULL, graph_uri TEXT DEFAULT NULL) RETURNS TABLE (node_iri TEXT, score FLOAT8)` SQL function; supported metrics: `'betweenness'`, `'closeness'`, `'eigenvector'`, `'katz'` (attenuation factor via `pg_ripple.katz_alpha` GUC, FLOAT8, default 0.01). Results stored in `_pg_ripple.centrality_scores (node BIGINT, metric TEXT, score FLOAT8, computed_at TIMESTAMPTZ, PRIMARY KEY (node, metric))`. `pg:centrality(?node, ?metric)` SPARQL function for inline lookup. PT0419 fires for unrecognised metrics; PT0420 fires on computation timeout.
- [x] **PR-TRUST-EIGEN-01** — Source-trust-weighted eigenvector centrality: `pg_ripple.centrality_run(..., metric = 'eigenvector_trust')` variant that seeds the power-iteration vector with per-node `pg:sourceTrust` values (from v0.87.0 PROV-CONF-01) rather than a uniform vector. Nodes whose triples derive predominantly from high-trust sources receive elevated initial authority mass, driving steady-state scores toward trust-backed authoritative entities. `pg:centrality(?node, 'eigenvector_trust')` SPARQL function. Stored in `_pg_ripple.centrality_scores` with `metric = 'eigenvector_trust'`. Falls back to standard eigenvector centrality with a WARNING when `prov_confidence = off` or no `pg:sourceTrust` triples are present. Requires v0.87.0 uncertain knowledge engine.
- [x] **PR-ENTITY-RESOLUTION-01** — Centrality-guided entity deduplication: `pg_ripple.pagerank_find_duplicates(metric TEXT DEFAULT 'betweenness', centrality_threshold FLOAT8 DEFAULT 0.1, fuzzy_threshold FLOAT8 DEFAULT 0.85) RETURNS TABLE (node_a_iri TEXT, node_b_iri TEXT, centrality_score FLOAT8, fuzzy_score FLOAT8)` SQL function. Combines v0.88.0 centrality scores from `_pg_ripple.centrality_scores` with v0.87.0 `pg:fuzzy_match()` to identify node pairs that are both structurally central (likely real entities) and textually similar (likely aliases). Results can be confirmed via `pg_ripple.assert_same_as(node_a TEXT, node_b TEXT)` which writes `owl:sameAs` triples and triggers OWL-sameAs canonicalization (v0.23.0); a follow-up `pagerank_run()` recomputes scores on the merged graph. REST endpoint `POST /pagerank/find-duplicates` streams results as JSON. Requires v0.87.0 fuzzy-matching functions and `centrality_run()` to have been executed first.
- [x] **PR-KATZ-TEMPORAL-01** — Temporal authority via Katz centrality: `pg_ripple.centrality_run(..., metric = 'katz_temporal', decay_rate FLOAT8 DEFAULT 0.01, temporal_predicate TEXT DEFAULT NULL)` computes Katz centrality with per-edge weight `exp(-decay_rate * age_in_days(t))` where `t` is the object of `(s <temporal_predicate> ?t)` (reusing the PR-TEMPORAL-01 temporal encoding). Edges without timestamps receive weight 1.0. Solves the "recent authority" problem: finds nodes that are both structurally important and currently active, making it suitable for citation graphs, news graphs, and live knowledge bases where old links should carry less weight. `pg:centrality(?node, 'katz_temporal')` SPARQL function. Stored in `_pg_ripple.centrality_scores` with `metric = 'katz_temporal'`. Requires `temporal_predicate` to be set; PT0415 fires for non-temporal objects (shared code path with PR-TEMPORAL-01).
