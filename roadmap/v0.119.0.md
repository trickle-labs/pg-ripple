# v0.119.0 — OWL propertyChainAxiom, SERVICE Circuit Breaker, and Schema-Aware NL→SPARQL

**Status:** ✅ Implemented

## Summary

v0.119.0 delivers three features from the Platform Maturity roadmap: `owl:propertyChainAxiom`
inference in the OWL-RL Datalog rule set (Feature 5), a federated SERVICE circuit
breaker with Prometheus gauge (Feature 6), and schema-aware NL→SPARQL translation
that includes active vocabulary bundle metadata in the LLM system prompt (Feature 10).
Also fixes the property-path-over-RDF-star gap and extends the OWL-RL regression suite.

## Deliverables

### Feature 5 — `owl:propertyChainAxiom` Inference

- [x] Implement `owl:propertyChainAxiom` in `src/datalog/builtins.rs` OWL-RL rule set
- [x] `?x owl:propertyChainAxiom (?p1 ?p2 … ?pn)` compiles to n-step Datalog chain rule
- [x] Add 10 canonical pg_regress examples:
  - FOAF `foaf:knows` chain (2-hop acquaintance)
  - SKOS `skos:broaderTransitive` closure
  - PROV-O `prov:wasInfluencedBy` derivation chain
  - 7 additional chain scenarios
- [x] Add propertyChainAxiom test to LUBM CI gate
- [x] Document in `docs/src/reference/owl-rl.md`

### Feature 6 — SERVICE Circuit Breaker

- [x] Create `_pg_ripple.federation_circuit_state (endpoint_iri TEXT PRIMARY KEY, state TEXT, last_failure_at TIMESTAMPTZ, failure_count INT)` table
- [x] Circuit trips to `'open'` after `pg_ripple.federation_circuit_failure_threshold` consecutive failures (default 5)
- [x] Transitions to `'half_open'` after `pg_ripple.federation_circuit_recovery_secs` seconds (default 60)
- [x] Returns to `'closed'` on first successful probe in `'half_open'` state
- [x] Add Prometheus gauge `pg_ripple_federation_circuit_state{endpoint}` (0=closed, 1=open, 2=half_open)
- [x] Migration: `CREATE TABLE _pg_ripple.federation_circuit_state (...)`

### Property Paths over RDF-star Gap Fix

- [x] Compile `?x ?p ?y` patterns inside `<<...>>` quote context to `temporal_facts`-style routing
- [x] Property paths traverse reified edges natively
- [x] Add 5 pg_regress tests for property paths over RDF-star

### OWL-RL sh:propertyChainAxiom Regression Extension

- [x] Extend `tests/pg_regress/sql/owl_rl_sameas.sql` with `owl:propertyChainAxiom` cycle-safety tests
- [x] Verify no infinite loop when a chain contains the derived property itself

### Feature 10 — Schema-Aware NL→SPARQL

- [x] Add GUC `pg_ripple.nl_sparql_include_bundles BOOLEAN DEFAULT on`
- [x] When on, `sparql_from_nl()` includes active vocabulary-bundle metadata in LLM system prompt:
  - SKOS preferred labels (`skos:prefLabel`)
  - DCTERMS titles (`dcterms:title`)
  - Schema.org type names (`schema:name`)
  - FOAF names (`foaf:name`)
- [x] Bundle metadata is fetched via SPI query before each `sparql_from_nl()` call
- [x] Document grounding improvement in `docs/src/cookbook/nl-to-sparql.md`
