# v0.115.0 — A16 Medium: HTTP API Parity and Observability

**Status:** ✅ Released

## Summary

v0.115.0 adds HTTP REST coverage for all major post-v0.111.0 subsystems that
lack it (M16-02), adds Prometheus metrics for the new subsystems (M16-03), fixes
SQL injection risk in pagerank handler (M16-04), documents the health/readiness
endpoint split (M16-09), adds a CI compat-check gate (M16-10), and secures the
`/metrics` endpoint with optional bearer-token authentication (M16-22).

## Deliverables

### M16-02 — HTTP Route Groups for New Subsystems

- [x] `GET/POST /temporal/{mark,point_in_time,facts}` endpoints
- [x] `POST /pprl/{bloom_encode,dice_similarity}` endpoints
- [x] `POST /dp/{noisy_count,noisy_histogram}` endpoints
- [x] `POST /entity-resolution/{resolve,evaluate,monitoring/{enable,disable}}` endpoints
- [x] `GET /proof-tree/{subject}/{predicate}/{object}` endpoint
- [x] `GET/POST/DELETE /tenants/{name}` endpoints
- [x] All write endpoints protected by `check_auth_write`
- [x] OpenAPI annotations for all new routes

### M16-09 — Health vs Readiness Documentation

- [x] Document `/health` (liveness: process alive) vs `/ready` (readiness: DB connection + schema version check) split

### M16-10 — CI Compat-Check Gate

- [x] Add `compat-check` job to `.github/workflows/release.yml`
- [x] Job asserts `COMPATIBLE_EXTENSION_MIN` is within one minor version of the current extension

- [x] Document in `pg_ripple_http/README.md`
